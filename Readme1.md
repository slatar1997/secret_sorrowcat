# 🗺️ Project "Will You Be My GF?" - London Trip & Watercolor Interactive Table Game (Vite + TS)

เว็บแอปพลิเคชันรูปแบบเกมเล่าเรื่อง (Storytelling / Mini-game) สำหรับเซอร์ไพรส์ขอเป็นแฟน (Private Project)
นำเสนอผ่านธีม "การเดินทางสู่ลอนดอน" ผสมผสานสิ่งของ 4 อย่างบนโต๊ะทำงานในมุมมองจากด้านบน (Top-down View) คุมโทนด้วยงานศิลปะสีน้ำที่อบอุ่นและโรแมนติก สร้างด้วย React + Vite เน้นเบา ลื่นไหล และทำเสร็จไว

## 🎨 Design Theme & Aesthetic (ธีมและดีไซน์)

  - **Heading & Titles (หัวข้อ/ตัวใหญ่):** ใช้ฟอนต์ `Itim` (ตัวหนามน ขี้เล่น เด่นชัดเตะตา)

### step 3 (Cinematic Sliding View – Implementation Spec)

- **หน้าจอเฟดเข้าภายในโบกี้รถไฟ (train cabin background คงที่)**  
- **ตรงกลางจอมี “กรอบหน้าต่างรถไฟ” เป็น element นิ่ง ไม่เคลื่อนไหว**  
- **กรอบหน้าต่างทำหน้าที่เป็น mask (ใช้ CSS overflow hidden) เพื่อซ่อนทุกอย่างที่อยู่นอกกรอบ**

- **ด้านหลังหน้าต่างมี container ของ “วิวลอนดอน” เป็นแถบแนวนอนยาว**
- **ภายใน container วางภาพเรียงแนวนอน 3 รูป:**
  - london-1.jpg (Big Ben)
  - london-2.jpg (แม่น้ำเทมส์)
  - london-3.png (Tower Bridge)

- **การเลื่อนภาพให้ใช้ Framer Motion**
- **animate ค่า transform: translateX จาก 0 ไปค่าติดลบ (เลื่อนไปทางซ้าย)**
- **ห้ามใช้ margin / position absolute แบบขยับ layout เพราะจะไม่ smooth**

- **ใช้ easing แบบ linear เพื่อให้ความรู้สึกเหมือนรถไฟเคลื่อนที่จริง**
- **กำหนด duration ยาว (ประมาณ 10–20 วินาที) เพื่อให้ cinematic**

- **หน้าต่างต้องนิ่ง 100% ห้าม animate**
- **animation ต้องเกิดที่ container ของภาพเท่านั้น (moving layer)**

- **เมื่อภาพเลื่อนครบทั้ง 3 ฉาก ให้หยุดที่ภาพสุดท้าย**
- **ค้างไว้ 0.5–1 วินาที**

- **จากนั้นใช้ Framer Motion onAnimationComplete (หรือ callback ของ animation)**
  - trigger transition ไป scene ถัดไป (desk scene)
- **transition ออกจาก scene ใช้ fade + blur + white flash เล็กน้อย (CSS transition / Framer Motion opacity + filter)**

 ### Step 4_1: Tortilla Mini-Game Specs & Folding Logic (ระบบมินิเกมตอติญ่าและการพับแป้ง)

1. State & Phase Logic Flow (ระบบจัดการสถานะและการควบคุมเฟส)
- เครื่องมือที่ใช้: React State (useState)
- ตรรกะการทำงาน:
  * ใช้ตัวแปรสถานะหลักควบคุมการเปลี่ยนเฟสของมินิเกมจาก เฟส 1 (หั่น) -> เฟส 2 (ลากวาง) -> เฟส 3 (พับแป้ง) และบันทึกผลลัพธ์เป็นตัวแปรสถานะปลายทาง isTortillaCooked = true เพื่อส่งค่ากลับไประบบใหญ่
  * เฟส 1 (Preparation): ตั้งตัวนับจำนวนการคลิก (Click Counter) แยกอิสระระหว่างชิ้นไก่และผักกาด เพื่อตรวจจับและสะสมการคลิกให้ครบชิ้นละ 5 ครั้งก่อนสั่งข้ามเฟส
  * เฟส 2 (Assembly): ทำฟังก์ชันคำนวณพิกัดจุดตัด (Collision Detection) รอบขอบเขตแผ่นแป้ง เมื่อผู้ใช้ปล่อยนิ้วลากวัตถุดิบลงในพิกัดเป้าหมาย ระบบจะเปลี่ยนสถานะวัตถุดิบเป็นล็อกตำแหน่ง (isDropped: true) และสั่งดูดเข้ากึ่งกลางแผ่นแป้งทันที เมื่อครบ 3 ชิ้นจะเปิดสวิตช์แสดงระบบลูกศรไกด์ไลน์เพื่อข้ามเฟส
  * เฟส 3 (Folding): ใช้ระบบตรวจสอบลำดับข้อมูลในอาร์เรย์ (Array Sequential Check) เพื่อบันทึกทิศทางการกดพับ โดยล็อกเงื่อนไขให้ทำงานเฉพาะลำดับ ล่าง -> ซ้าย -> ขวา -> บน เท่านั้น หากกดผิดลำดับระบบจะไม่ทำงาน

2. Asset & UI Layout Design (การจัดวางโครงสร้างและเลเยอร์หน้าจอ)
- เครื่องมือที่ใช้: Tailwind CSS 100%
- ตรรกะการทำงาน:
  * เรียกใช้งานรูปภาพทั้งหมดจากพาธ public/images/tortilla/ ประกอบด้วยภาพ tortilla.png, chicken.png, lettuce.png, และ sauce.png
  * ใช้ระบบ Flexbox หรือ Grid จัดพิกัดหน้าจอแบ่งเป็นถาดสล็อตวัตถุดิบด้านข้าง และพื้นที่เขียงแผ่นแป้งหลักที่คุมตำแหน่งแบบกึ่งกลางจอ (relative)
  * โครงสร้างเลเยอร์การพับแป้ง (Phase 3 Layering): จัดการลำดับความสูงของวัตถุดิบด้วยระบบ Z-Index แยกชิ้นส่วนแผ่นแป้งออกเป็นส่วนปีกพับ 4 ทิศทาง โดยตั้งค่าระดับเลเยอร์ของปีกแป้งให้อยู่สูงกว่าระดับของวัตถุดิบด้านใน (วัตถุดิบอยู่ชั้นล่าง -> ปีกแป้งอยู่ชั้นบน) เพื่อให้ชิ้นแป้งเคลื่อนที่ขึ้นมาถมทับปิดบังเนื้อสัตว์และผักได้มิดชิด

3. Animation & Interaction Mapping (ระบบควบคุมการเคลื่อนไหวและเอฟเฟกต์)
- เครื่องมือที่ใช้: Framer Motion และ Canvas Confetti
- ตรรกะการทำงาน:
  * เฟส 1 (หั่นวัตถุดิบ): ใช้คำสั่งจำลองการกดปุ่มยุบตัว (whileTap) ขนาด scale: 0.95 ในทุกจังหวะคลิก และเมื่อคลิกครบ 5 ครั้ง ให้สั่งเล่นอนิเมชันแตกกระจายตัวออกจากกัน ควบคู่กับการลดค่าความโปร่งแสง (Opacity) เป็น 0 เพื่อเปิดทางให้วัตถุดิบชิ้นถัดไป
  * เฟส 2 (ระบบลากวางคอมโพเนนต์): เปิดระบบคำสั่งลาก (drag) บนชิ้นส่วนวัตถุดิบ และตั้งค่าตัวแปรแรงดีดกลับ (dragTransition) ด้วยค่าความแข็งของสปริงสูงเพื่อให้วัตถุดิบเด้งกลับไปที่จุดเริ่มต้นทันทีหากปล่อยนอกพื้นที่แผ่นแป้ง และใช้ฟังก์ชันล็อกพิกัดหยุดระบบลากทันทีเมื่อเข้าพื้นที่ถูกต้อง
  * เฟส 3 (แอนิเมชันพับ 3D ทิศทาง): เมื่อมีการคลิกปุ่มพับแต่ละด้านตามลำดับที่กำหนด สั่งให้ชิ้นส่วนปีกแป้งด้านนั้นเล่นอนิเมชันหมุนพลิกแกน 3 มิติ (คำสั่ง rotateX สำหรับด้านล่าง/บน และ rotateY สำหรับด้านซ้าย/ขวา) ควบคู่กับคำสั่งขยับตำแหน่งเลื่อน (translate) ตลบเข้าหากึ่งกลางแผ่นแป้ง
  * เฟสความสำเร็จ (Success Event): เมื่อระบบตรวจพบการพับครบด้านสุดท้าย ชิ้นแป้งห่อสมบูรณ์จะเล่นอนิเมชันกระเด้งตัวแนวดิ่งหนึ่งครั้งด้วยแกน Y ควบคู่กับการเรียกใช้งานไลบรารีภายนอกสั่งยิงพาร์ทิเคิลพลุรูปหัวใจ (canvas-confetti) กระจายออกรอบแผ่นแป้ง ก่อนจะใช้คำสั่งลดค่าความโปร่งแสงเฟดจางหายไป (Fade Out) เพื่อปิดหน้าต่างมินิเกมและตัดกลับสู่กระดานโต๊ะทำงานหลัก
  
### Step4_2 Vinyl Station Interaction & Audio Workflow (กระบวนการทำงานของเครื่องเล่นแผ่นเสียง 12 เพลง)

**1. การควบคุมสถานะระบบ (System Tracking)**
- ระบบจะคอยจำว่าตอนนี้แผ่นเสียงแผ่นไหน (เลข 1 ถึง 12) กำลังถูกเปิดใช้งานอยู่ หรือยังไม่มีการเลือกแผ่น
- ระบบจะเช็กสถานะการทำงานว่าเพลงกำลังเล่นหรือหยุดอยู่ เพื่อนำไปสั่งให้แผ่นเสียงเริ่มหมุนหรือหยุดหมุน และควบคุมให้ก้านหัวเข็มยกขึ้นหรือวางลงบนแผ่น
- มีระบบตรวจจับตำแหน่งเมื่อผู้ใช้ลากแผ่นเสียงเข้ามาใกล้ ๆ บริเวณตัวเครื่องเล่น เพื่อเตรียมเปิดโซนรับแผ่น

**2. กระบวนการย้ายและสลับแผ่นเสียง (Drag & Drop / Click Swap)**
- **กล่องเก็บแผ่นเสียง:** แผ่นเสียงทั้ง 12 แผ่นจะนอนอยู่ในกล่อง แฟนสามารถใช้นิ้วลากแผ่นที่ต้องการขยับไปมาบนจอได้อย่างอิสระ
- **เมื่อลากมาวางบนเครื่องเล่นสำเร็จ:** - ระบบจะทำการสลับแผ่นให้อัตโนมัติ โดยแผ่นเสียงอันเก่าที่เคยเล่นอยู่ก่อนหน้านี้ จะเล่นอนิเมชันลอยดึ๋งเด้งกลับเข้าไปเก็บเข้าที่ในกล่องของตัวเองทันที ส่วนแผ่นใหม่จะดูดติดเข้าล็อกกึ่งกลางเครื่องเล่นแทน
- **ระบบกดจิ้มทางเลือก:** หากแฟนไม่ได้ลาก แต่ใช้วิธีกดจิ้มที่แผ่นเสียงในกล่อง ตัวแผ่นนั้นจะเล่นอนิเมชันลอยสไลด์ข้ามจอจากกล่องมาวางลงล็อกบนเครื่องเล่นให้เองโดยอัตโนมัติ

**3. กระบวนการทำงานของระบบเสียง (Audio & Background Play Workflow)**
- **ระบบเสียงเชื่อมโยงทั้งเว็บ:** ตัวจัดการเสียงจะถูกตั้งค่าไว้ที่โครงสร้างส่วนบนสุดของระบบ เพื่อทำให้เพลงสามารถเล่นคลอเป็นพื้นหลัง (Background Music) ได้ยาว ๆ อย่างต่อเนื่อง ไม่มีอาการเสียงดับหรือกระตุก แม้ว่าแฟนจะกดปิดหน้าต่างเครื่องเล่นเสียงนี้เพื่อไปเปิดดูสมุดภาพหรือเล่นมินิเกมทำอาหารชิ้นอื่นบนโต๊ะก็ตาม
- **ลำดับขั้นตอนการเปลี่ยนเพลง (Playback Sequence):**
  - เมื่อมีการเปลี่ยนแผ่นเสียง ➡️ ระบบจะสั่งหยุดเล่นเพลงเก่าทันที ➡️ โหลดไฟล์เสียงของเพลงใหม่มารอ ➡️ ก้านหัวเข็มหมุนเลื่อนมาวางแปะบนขอบแผ่นเสียง ➡️ ดีเลย์จังหวะเล็กน้อยให้ดูสมจริง ➡️ เริ่มเล่นไฟล์เพลงใหม่ พร้อมสั่งให้แผ่นไวนิลสีดำและรูปคู่ตรงกลางเริ่มหมุนติ้ว ๆ พร้อมกัน

**4. เทคนิคการคุมภาพหน้าปกตรงกลางแผ่นเสียงไม่ให้ยืดและเบี้ยว (UI Center Scaling)**
- ให้ AI จัดวางบล็อกรูปภาพหน้าปกคู่ไว้ตรงจุดกึ่งกลางของแผ่นเสียงสีดำเป๊ะ ๆ ด้วยคำสั่งจัดกึ่งกลางสัมบูรณ์ (Absolute Centering)
- **คีย์เวิร์ดล็อกรูปภาพ:** ตัวไฟล์รูปภาพหน้าปก (`cover-01.png` ถึง `cover-12.png`) จะต้องถูกบังคับด้วยคำสั่ง `object-cover` และ `rounded-full` เสมอ เพื่อให้ตัวบราวเซอร์ทำการบีบอัดและตัดขอบรูปภาพดุมกลางให้เป็นวงกลม 100% โดยอัตโนมัติ โดยที่สัดส่วนภาพ (Aspect Ratio) ของรูปคู่ต้นฉบับจะยังคงเดิม ไม่มีการยืด ขยายจนเบี้ยว หรือบี้แบน ไม่ว่ารูปจริงที่โยนลงไปจะเป็นสี่เหลี่ยมจัตุรัส แนวตั้ง หรือแนวนอนก็ตาม

**5. กระบวนการทำงานของปุ่มควบคุมการเล่นเพลง (Play / Pause / Stop Workflow)**
- มีการเพิ่มแผงปุ่มกดสไตล์มินิมอล 3 ปุ่ม (Play, Pause, Stop) ไว้บริเวณตัวเครื่องเล่นแผ่นเสียง
- **ปุ่ม Play (เล่นเพลง):** - กดได้ต่อเมื่อมีแผ่นเสียงวางอยู่บนเครื่องแล้วเท่านั้น 
  - เมื่อกด: เพลงจะเริ่มเล่น (หรือเล่นต่อจากที่หยุดชั่วคราว), ก้านหัวเข็มจะเลื่อนมาวางบนแผ่น, และแผ่นไวนิลพร้อมรูปคู่จะเริ่มหมุนติ้ว ๆ
- **ปุ่ม Pause (หยุดชั่วคราว):** - เมื่อกด: เพลงจะหยุดนิ่งอยู่กับที่ ณ วินาทีนั้น, แผ่นไวนิลและรูปคู่จะหยุดหมุนทันที, แต่ก้านหัวเข็มยังคงวางคาอยู่บนแผ่นเสียงเหมือนเดิม (ไม่ดีดกลับ) เพื่อรอสลับกลับมาเล่นต่อ
- **ปุ่ม Stop (หยุดสนิท):** - เมื่อกด: เพลงจะหยุดเล่นและถูกรีเซ็ตกลับไปเริ่มต้นนาทีที่ 0:00 ใหม่ทั้งหมด, แผ่นไวนิลและรูปคู่หยุดหมุน, และก้านหัวเข็มจะยกตัวลอยขึ้นแล้วเลื่อนหมุนกลับไปเก็บ ณ แท่นวางของตัวเองอัตโนมัติ
    

  ## step4_3 Memory Diary Logic & Interaction (ตรรกะระบบสมุดภาพความทรงจำมุมมอง Top-View)

**1. Visual & Layout (2D Top-Down Book View)**
- **Top-Down Perspective:** ตัวสมุดบันทึกจะวางแบนราบอยู่บนโต๊ะทำงานในมุมมองมองตรงจากด้านบน (เหมือนเราก้มหน้าอ่านหนังสือที่วางอยู่บนโต๊ะ)
- **Grid Pattern หน้าละ 4 รูป:** เนื้อหาข้างในสมุดแต่ละหน้า (Page 1 ถึง Page 5) จะจัดเลย์เอาต์เป็นตาราง Grid 2x2 เพื่อวางรูปถ่ายโพลารอยด์หน้าละ 4 ใบ และมีพื้นที่สำหรับพิมพ์ข้อความบรรยายความทรงจำสั้น ๆ ไว้ใต้รูปแต่ละใบ

**2. Assets Mapping Logic (ตรรกะการกระจายรูป 20 ใบ)**
- **โฟลเดอร์เก็บรูป:** `public/images/diary/` โดยมีไฟล์รูปชื่อ `photo-01.png` ไปจนถึง `photo-20.png`
- **ตรรกะการแบ่งหน้า (Pagination Array):** - สั่งให้ AI ทำการตัดแบ่ง (Slice) อาร์เรย์รูปภาพทั้ง 20 ใบ ออกเป็นชุด ชุดละ 4 รูป เพื่อเอาไปกระจายลงตามหน้าสมุดทั้ง 5 หน้าโดยอัตโนมัติ:
    - หน้าที่ 1 ➡️ แสดงรูป `photo-01` ถึง `photo-04`
    - หน้าที่ 2 ➡️ แสดงรูป `photo-05` ถึง `photo-08`
    - หน้าที่ 3 ➡️ แสดงรูป `photo-09` ถึง `photo-12`
    - หน้าที่ 4 ➡️ แสดงรูป `photo-13` ถึง `photo-16`
    - หน้าที่ 5 ➡️ แสดงรูป `photo-17` ถึง `photo-20`

**3. Interaction & Animation Logic (ระบบเปิดหน้ากระดาษ 3D)**
- **การคุมไลบรารี `react-pageflip`:** ตัวสมุดจะรองรับการใช้นิ้วปัด (Swipe) บนมือถือ หรือใช้เมาส์คลิกแล้วลากเพื่อพลิกหน้ากระดาษ โดยปีกกระดาษจะโค้งงอตามแรงลากในมุมมองแบบ 3D สมจริง แต่ตัวเลย์เอาต์ภาพรวมยังคงระนาบอยู่ในมุมมอง Top-down 
- **การจำกัดพื้นที่แสดงผล:** รูปถ่ายคู่ทั้ง 20 ใบจะถูกล็อกสัดส่วนไว้ ไม่ให้เบี้ยวหรือล้นขอบกระดาษ และรูปทั้งหมดในสมุดนี้จะถูกเซ็ตให้พุ่งกระจายระเบิดออกมาสมทบร่วมกับไอเทมอื่น ๆ ตอนที่แฟนเปิดฝากล่องของขวัญในฉากจบ (Grand Finale)
    

  ### Step4_4 Secret Box & Quiz Interaction Workflow (กระบวนการทำงานของกล่องปริศนาและระบบล็อกคำถาม)

**1. เงื่อนไขการปลดล็อกกล่องบนโต๊ะหลัก (Lock & Unlock Conditions)**
- **สถานะเริ่มต้น (Locked):** ตัวกล่องของขวัญ (`box.png`) จะวางอยู่บนโต๊ะทำงานในมุมมอง Top-down โดยมีไอคอนแม่กุญแจสีเทาทับอยู่ แฟนจะไม่สามารถคลิกเข้าไปเล่นได้
- **เงื่อนไขการปลดล็อกปุ่ม:** ระบบจะคอยเช็กว่าแฟนทำกิจกรรมอื่นบนโต๊ะครบถ้วนแล้วหรือยัง (เคลียร์มินิเกมตอติญ่าสำเร็จ + เปิดฟังแผ่นเสียงไวนิลแล้ว + เปิดอ่านสมุดภาพความทรงจำครบถ้วนแล้ว)
- **เมื่อเงื่อนไขครบ (Unlocked):** แม่กุญแจจะเปลี่ยนเป็นสีทองอร่าม พร้อมเล่นเสียงเอฟเฟกต์การปลดล็อก (`unlock.mp3`) เพื่อเป็นสัญญาณบอกแฟนว่ากล่องปริศนาพร้อมให้กดเข้าไปเล่นแล้ว

**2. กระบวนการเล่นมินิเกมคำถามรูปภาพ (Quiz Phase)**
- เมื่อแฟนลากกุญแจคลิกที่กล่อง ระบบจะเปิดหน้าต่างป๊อปอัปคำถามขึ้นมา มีทั้งหมด **2 ข้อใหญ่**
- แต่ละข้อจะแสดงหัวข้อคำถาม และมีตัวเลือกคำตอบเป็นรูปภาพสี่เหลี่ยมสีน้ำจำนวน **4 ตัวเลือก** (ดึงรูปจากโฟลเดอร์ `quiz` ตามพิกัด `q1-1.png` ถึง `q1-4.png` และ `q2-1.png` ถึง `q2-4.png`) จัดเรียงเป็นตะแกรงสี่ช่องพอดีจอมือถือ
- **ตรรกะการตอบคำถาม:** - เลือกอะไรก็ได้เมื่อเลือกแล้วเปลี่ยนไปคำถามข้อที่ 2 ทันที


**3. กระบวนการรับรางวัลหลังตอบถูกครบ (Reward Phase)**
- เมื่อแฟนตอบคำถามรูปภาพถูกครบทั้ง 2 ข้อใหญ่ หน้าต่างป๊อปอัปคำถามจะปิดลง
- ระบบจะส่งภาพลูกกุญแจสีน้ำวินเทจ (`key.png`) ให้ลอยฟึ่บขึ้นมาวางเด่นอยู่บนโต๊ะทำงานถัดจากตัวกล่อง เพื่อให้แฟนสามารถลากลูกกุญแจนี้ไปใช้ทำพิธีไขเปิดตัวกล่องและฝากล่อง (`pha.png`) ในฉากจบ (Grand Finale) ต่อไป

    ### Step4_4_1 Quiz Selection & Grand Finale State Flow (ตรรกะการส่งค่ารูปควิซไปฉากจบ)

**1. State Tracking (App Component)**
- สร้าง global state หรือ pass props เพื่อเก็บไฟล์รูปภาพที่แฟนเลือกตอบจากมินิเกมคำถามทั้ง 2 ข้อ
- ตัวอย่างตัวแปร: `selectedQuizImages = [chosen_q1_path, chosen_q2_path]` (รูปภาพมาจากโฟลเดอร์ `public/images/minigame/`)

**2. Data Flow & Component Interaction**
- **ตอนเล่นควิซ:** เมื่อแฟนกดเลือกคำตอบในข้อ 1 และข้อ 2 ให้ระบบบันทึก Path รูปภาพใบที่เลือกนั้นลงไปใน State ทันที
- **ตอนฉากจบ (Finale):** เมื่อแฟนลากกุญแจมาไขกล่องของขวัญสำเร็จ (ฝากล่อง `pha.png` แยกตัวออก) ให้ดึงรูปคู่ทั้ง 2 ใบที่ถูกบันทึกไว้ใน `selectedQuizImages` พุ่งกระจาย (Explode) ออกมาจากกล่องร่วมกับไอเทมอื่น ๆ ทันที
- **Interaction:** รูปคู่จากควิซที่พุ่งออกมา ต้องใส่คำสั่งของ `framer-motion` ให้แฟนสามารถใช้เมาส์หรือนิ้วลากเกลี่ยไปมาบนจอได้อย่างอิสระ (Drag Sandbox)

### 💖 Step 5: Grand Finale & Interactive Sandbox Workflow (กระบวนการทำงานฉากจบ)

**1. พิธีไขกุญแจและเปิดฝากล่อง (Lockpicking & Manual Open)**
- **การลากกุญแจ:** แฟนต้องทำการลากลูกกุญแจ (`key.png`) บนหน้าจอ ไปหย่อน (Drop) ให้ตรงตำแหน่งของตัวกล่อง (`box.png`)
- **ระบบเลื่อนฝากล่องด้วยมือ:** เมื่อกุญแจลงล็อก ระบบจะปลดล็อกให้แฟนสามารถใช้นิ้วหรือเมาส์ "คลิกลาก" ตัวฝากล่อง (`pha.png`) ให้เลื่อนแยกออกจากตัวกล่องท่อนล่างได้จริง ๆ เมื่อลากฝาพ้นขอบกล่องแล้ว ฝากล่องจะค่อย ๆ เฟดจางหายไปเพื่อเปิดทางให้ไอเทมด้านในพุ่งออกมา

**2. เอฟเฟกต์ระเบิดความทรงจำและพื้นที่อิสระ (Memory Burst & Sandbox)**
- **จังหวะระเบิด (Explode):** ทันทีที่ฝากล่องถูกเปิดออก ไอเทมทั้งหมดที่สะสมมาจากทุกด่านจะพุ่งกระจายตัวลอยออกมาจากกลางกล่องพร้อม ๆ กันแบบสุ่มทิศทางทั่วหน้าจอ ประกอบไปด้วย:
  - แผ่นแป้งตอติญ่าที่ทำเสร็จจากมินิเกม
  - รูปถ่ายคู่จากโพลารอยด์ในสมุดภาพไดอารี่ทั้งหมด 20 ใบ (`photo-1.png` ถึง `photo-20.png`)
  - คำตอบ ที่แฟนเลือกตอบจากมินิเกมกล่องปริศนา (Quiz)
- **พื้นที่ลากเล่น(Interactive Sandbox):** ไอเทมความทรงจำทุกชิ้นที่ลอยกระจายอยู่บนจอ แฟนสามารถใช้เมาส์คลิกหรือใช้นิ้วจิ้มเพื่อลากเกลี่ย ย้ายตำแหน่ง ปัดกองรูป หรือจัดวางใหม่บนพื้นที่หน้าจอได้อย่างอิสระตลอดเวลา

**3. จดหมายลับและระบบปุ่มกดขอเป็นแฟน (The Secret Letter & Button Chaos)**
- **เปิดจดหมาย:** หลังกองรูปพุ่งกระจายออกไป ที่ก้นกล่องจะปรากฏรูปซองจดหมายลับ (`envelope.png`) เมื่อแฟนกดคลิก จะมีแอนิเมชันกระดาษจดหมายเด้งขยายใหญ่ขึ้นมาตรงกลางจอ
- **เอฟเฟกต์พิมพ์ดีด:** ข้อความสารภาพรักจะค่อย ๆ พิมพ์ตัวเองขึ้นมาทีละตัวอักษรอย่างโรแมนติกว่า **"เป็นแฟนกันได้มั้ย"** พร้อมปรากฏปุ่มให้เลือก 2 ทางเลือก: **[ เป็น ]** และ **[ ไม่เป็น ]**
- **พฤติกรรมปุ่ม [ ไม่เป็น ] (ปุ่มแกง):** - หากแฟนพยายามเอาเมาส์ไปชี้หรือเอานิ้วไปจิ้มโดน ปุ่มนี้จะสั่งให้หน้าจอเกิดการสั่นเตือน (Screen Shake)
  - จากนั้นตัวปุ่มจะสุ่มวาร์ปย้ายตำแหน่งหนีไปทั่วหน้าจอทันที ทำให้แฟนกดไม่ได้หรือกดโดนยากมาก
- **พฤติกรรมปุ่ม [ เป็น ] (ปุ่มล็อกมง):**
  - ปุ่มนี้สามารถกดได้ตามปกติอย่างเสถียร
  - **ระบบขยายใหญ่ตามเวลา:** ตั้งแต่จดหมายเปิดออก ปุ่มนี้จะค่อย ๆ สเกลขยายขนาดตัวเองให้ใหญ่ขึ้นเรื่อย ๆ อย่างช้า ๆ ตามเวลาที่ผ่านไป (ไปสิ้นสุดขนาดใหญ่ที่สุดเมื่อเวลาครบ 1 นาที) เพื่อเพิ่มโอกาสและดึงดูดสายตาให้แฟนกดปุ่มนี้ให้ได้
- **เมื่อกดปุ่ม [ เป็น ] สำเร็จ:** ระบบจะจุดพลุหัวใจระเบิดฟู่ออกมาเต็มหน้าจอแบบชุดใหญ่ไฟกระพริบเพื่อเฉลิมฉลองการตกลงเป็นแฟนกันสำเร็จ

## 🛠️ UI/UX & Tech Stack (เครื่องมือที่ใช้พัฒนา)

- **Core Engine & Build Tool:** `React + Vite` - โครงสร้างหลักของเว็บ เบาหวิว รันเร็ว พัฒนาได้คล่องตัวสูง
- **Language:** `TypeScript (.ts / .tsx)` - ล็อกชนิดข้อมูลและควบคุมความปลอดภัยของโค้ด ป้องกันการเกิด Error
- **Styling & Layout:** `Tailwind CSS` - จัดตำแหน่งสิ่งของบนโต๊ะด้วยพิกัด Absolute Positioning และคุมโทนสีพาสเทลสีน้ำ
- **UI Components:** `daisyui` - ทำโครงสร้างแป้นพิมพ์รหัสผ่าน, ป๊อปอัปคำถาม และเพิ่มเอฟเฟกต์แสงฟุ้งๆ
- **Interactions & Animation:**
  - `Framer Motion` - ระบบ Drag & Drop แผ่นเสียง, วัตถุดิบอาหาร, ลูกกุญแจ, แอนิเมชันตั๋วพลิก และเอฟเฟกต์รูปกระจายฉากจบ
  - `React PageFlip` - จำลองแอนิเมชันเปิดพลิกหน้าสมุดบันทึกแบบ 3D สมจริง
- **Sound Elements:** `HTML5 Audio Context / Use-Sound` - จัดการระบบเสียงเอฟเฟกต์ในเกม และควบคุมเพลงเบื้องหลังไม่ให้ดับ
- **Hosting:** `Netlify` - เชื่อมต่อกับคลังโค้ดบน `GitHub` เพื่อปล่อยเว็บออนไลน์อัตโนมัติ
- **Heading & Titles (หัวข้อ/ตัวใหญ่):** ใช้ฟอนต์ `Itim` (ตัวหนามน ขี้เล่น เด่นชัดเตะตา)
- **Body Text (เนื้อหาทั่วไป/ซับไตเติล):** ใช้ฟอนต์ `Mali` (ลายมือน่ารัก เรียบร้อย อบอุ่น รองรับทั้งไทยและอังกฤษ)

📜 Coding Rules (กฎในการเขียนโค้ด)
- ใช้ TypeScript ทุกไฟล์: ห้ามหลุดไปใช้ไฟล์ตระกูล .js หรือ .jsx เด็ดขาด
= หลีกเลี่ยง any ถ้าไม่จำเป็น: พยายามระบุชนิดข้อมูล (Type/Interface) ให้ชัดเจนเสมอเพื่อความปลอดภัย
- All UI styling must use Tailwind utility classes. 
- Avoid inline style and custom CSS unless required for global setup (e.g. fonts).
- แยก Component ให้เล็กและอ่านง่าย: แตกย่อยของเล่นแต่ละชิ้นออกจากกันอย่างชัดเจนตามผังโฟลเดอร์
- เน้นอ่านง่ายมากกว่าสั้นเกินไป: เขียนโค้ดที่สื่อความหมายชัดเจน เพื่อไม่ให้สับสนในภายหลัง
- ห้ามลบโค้ดเดิมถ้าไม่เข้าใจหน้าที่: ห้ามตัดโค้ดทิ้งสุ่มสี่สุ่มห้าเด็ดขาด หากยังไม่ได้ตรวจสอบว่าทำหน้าที่อะไร
- ห้ามจำลองการรันโปรแกรม หรือเดาว่ารันแล้วจะเกิดอะไรขึ้น
