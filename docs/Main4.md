### 🪵 Step 4: Central Memory Table Specs & Interactive Hub (ฉากหลัก - โต๊ะความทรงจำสีน้ำ)

1. State & Logic Flow (ระบบจัดการสถานะและตรรกะตัวแปร)
- เครื่องมือที่ใช้: React State (useState) หรือตัวควบคุมการนำทาง (Navigation Routing)
- ตรรกะการทำงาน:
  * ใช้ตัวแปรสถานะหลักควบคุมหน้าจอหลัก (ด่านโต๊ะทำงานมุมมอง Top-down) เพื่อทำหน้าที่เป็นจุดศูนย์กลาง (Hub) ในการกระจายงานไปมินิเกมย่อยทั้ง 4 ด่าน
  * เมื่อแฟนกดคลิกที่ปุ่มใดปุ่มหนึ่ง ระบบจะทำการสลับค่าสถานะด่านปัจจุบันเพื่อนำทางข้ามเข้าสู่ขั้นตอนย่อยทันที (เช่น กดปุ่มแรก -> เปลี่ยนสถานะเพื่อเปิดหน้าต่าง Step 4_1)
  * มีระบบตรวจสอบการบันทึกสถานะปลายทางย้อนกลับ (Backward State Check) เช่น หากกลับมาจาก Step 4_1 แล้วค่า `isTortillaCooked` เปลี่ยนเป็น true ระบบจะนำค่านั้นมาปรับเปลี่ยนหน้าตาหรือเปิดสัญลักษณ์เคลียร์ด่านบนปุ่มตอติญ่าเพื่อบันทึกประวัติความคืบหน้า

2. UI Design & Layout (การจัดวางโครงสร้างและเลเยอร์หน้าจอด้วย Tailwind CSS)
- เครื่องมือที่ใช้: Tailwind CSS 100%
- ตรรกะการทำงาน:
  * ออกแบบพื้นหลังเป็นแผ่นไม้หรือโต๊ะทำงานในมุมมองจากบนลงล่าง (Top-down view) คุมโทนสีน้ำตาลเนื้อไม้อ่อนตามข้อกำหนดดีไซน์หลัก
  * ระบบตอบสนองตามขนาดหน้าจอ (Responsive Scaling): ใช้คลาสควบคุมสัดส่วนขอบเขต (Container Max-Width) ร่วมกับการแปลงสเกลวัตถุ เพื่อให้ตัวโต๊ะทำงานและแผงปุ่มกดทั้งหมดหดขนาดย่อส่วนลงมาพอดีกับหน้าจอมือถือโดยอัตโนมัติ โดยที่องค์ประกอบไม่หลุดขอบและไม่ต้องเลื่อนหน้าจอแนวตั้ง
  * การจัดวางแผงควบคุมหลัก: จัดวางปุ่มเลือกด่านทั้ง 4 ปุ่มให้อยู่กึ่งกลางหน้าจอในรูปแบบตาราง 2 คอลัมน์ x 2 แถว (Grid Grid-cols-2 Gap-4) เพื่อความสมดุลและง่ายต่อการใช้นิ้วโป้งจิ้มบนโทรศัพท์มือถือ
  * โครงสร้างไอคอนและข้อมูลปุ่ม (Button Mapping):
    * ปุ่มที่ 1: Tortilla (ตอติญ่า) -> แสดงข้อความพร้อมไอคอน/อิโมจิตอติญ่า สั่งทางไป Step 4_1 (มินิเกมพับแป้ง)
    * ปุ่มที่ 2: Vinyl (แผ่นเสียง) -> แสดงข้อความพร้อมไอคอน/อิโมจิหูฟัง สั่งทางไป Step 4_2 (เครื่องเล่นเพลงไวนิล)
    * ปุ่มที่ 3: Diary (ไดอารี่) -> แสดงข้อความพร้อมไอคอน/อิโมจิสมุดบันทึก สั่งทางไป Step 4_3 (สมุดความทรงจำ)
    * ปุ่มที่ 4: Box (กล่องของขวัญ) -> แสดงข้อความพร้อมไอคอน/อิโมจิกล่องของขวัญ สั่งทางไป Step 4_4 (ฉากจบ Grand Finale)

3. Animation & Interaction Mapping (ระบบควบคุมการเคลื่อนไหวและเอฟเฟกต์)
- เครื่องมือที่ใช้: Framer Motion
- ตรรกะการทำงาน:
  * แอนิเมชันเปิดฉากโต๊ะทำงาน (Fade-in Hub): ทันทีที่เปลี่ยนฉากมาจากระบบโบกี้รถไฟวิ่งใน Step 3 ให้เลเยอร์โต๊ะทำงานและปุ่มทั้ง 4 ค่อยๆ คลายความโปร่งแสงเฟดขึ้นมาพร้อมกันอย่างนุ่มนวล
  * เอฟเฟกต์การโต้ตอบปุ่มกด (Micro-interactions): ใส่คำสั่งขยายขนาดเล็กน้อยเมื่อเมาส์ชี้ผ่านบน Desktop (`whileHover: { scale: 1.05 }`) และคำสั่งจำลองแรงกดปุ่มยุบตัวลงเมื่อแฟนใช้นิ้วจิ้มสัมผัสผ่านมือถือ (`whileTap: { scale: 0.95 }`) เพื่อให้ปุ่มดูมีมิติการตอบสนองที่จับต้องได้จริง
  * อนิเมชันปุ่มสั่นไกด์ไลน์: หากแฟนยังเลือกไม่ถูกว่าจะกดปุ่มไหนภายใน 5 วินาทีแรก สามารถสั่งให้ปุ่มที่ยังไม่เคยถูกกด เล่นอนิเมชันขยับดึ๋งเบาๆ เพื่อดึงดูดสายตาและนำทางให้แฟนรู้ว่าสามารถใช้นิ้วจิ้มลงไปบนปุ่มเหล่านี้ได้
