### 🎁 Step 4_4 & 4_4_1: Secret Box, Quiz Interaction & Grand Finale State Flow (ระบบกล่องปริศนา ควิซรูปภาพ และตรรกะฉากจบ)

1. State & Logic Flow (ระบบจัดการสถานะและการควบคุมเงื่อนไขล็อกกล่อง)
- เครื่องมือที่ใช้: React State (useState) หรือ React Context API (สำหรับแชร์ข้อมูล Global State)
- ตรรกะการทำงาน:
  * สถานะเริ่มต้นล็อกกล่อง (Box Locked State): ตั้งค่าตัวแปรควบคุมสิทธิ์การเปิดกล่องปริศนาเป็น Boolean (เช่น isBoxUnlocked: false) โดยระบบจะดักฟังประวัติการเคลียร์ด่านอื่นบนโต๊ะทำงานหลักแบบ Real-time คือ ต้องผ่านมินิเกมตอติญ่าแล้ว (`isTortillaCooked === true`) + เปิดฟังแผ่นเสียงไวนิลแล้ว + และเปิดอ่านสมุดบันทึกครบถ้วนแล้ว สิทธิ์จึงจะถูกเปลี่ยนเป็น true
  * ระบบบันทึกคำตอบควิซ (Global Quiz Selection State): สร้างชุดข้อมูลตัวแปรประเภท Array บนโครงสร้างระดับบนสุด (App Component/Root Context) ชื่อ `selectedQuizImages` เพื่อทำหน้าที่เก็บค่า String Path ของไฟล์รูปภาพคำตอบ 2 ใบที่แฟนเลือกจิ้มมาจากระบบคำถามควิซภาพ
  * ตรรกะการดำเนินเนื้อเรื่องควิซ (Quiz Progression Logic): ตั้งตัวนับลำดับข้อคำถาม (เช่น quizStep: number) เริ่มต้นที่ 1 สู่ข้อที่ 2 โดยมีข้อกำหนดพิเศษคือ ตัวระบบคำถามนี้จะไม่มีการตรวจผลผิด-ถูก (เลือกช้อยส์ใดก็ได้ถือว่าผ่านเกณฑ์และบันทึกค่านั้นลงใน Array ทันที) เมื่อแฟนกดเลือกคำตอบในข้อที่ 1 ระบบจะ Push ค่าพิกัดรูปภาพลง Array และปรับสถานะเปลี่ยนข้ามไปแสดงคำถามข้อที่ 2 ทันที และเมื่อเลือกข้อที่ 2 เสร็จสิ้น จะสั่งปิดหน้าต่างควิซและสั่งเข้าสู่เฟสรับรางวัลทันที

2. Asset & UI Layout Design (การจัดวางโครงสร้างเลเยอร์หน้าจอและสล็อตควิซ)
- เครื่องมือที่ใช้: Tailwind CSS 100%
- ตรรกะการทำงาน:
  * พิกัดเรียกใช้ไฟล์ทรัพยากรกราฟิก: ตัวกล่องหลักอ้างอิงจาก `box.png` ฝากล่องแยกชิ้นอ้างอิงจาก `pha.png` ลูกกุญแจสีน้ำวินเทจอ้างอิงจาก `key.png` (ทั้งหมดอยู่ในโฟลเดอร์ `public/images/`) และรูปภาพตัวเลือกควิซข้อที่ 1-2 อ้างอิงจากพาธ `public/images/quiz/` (ไฟล์ `q1-1.png` ถึง `q1-4.png` และ `q2-1.png` ถึง `q2-4.png`)
  * เลเยอร์ปุ่มกล่องบนโต๊ะหลัก: แสดงผลรูปกล่องของขวัญจัดวางบนพิกัดสัมพัทธ์ (Relative) บนโต๊ะทำงานมุมมอง Top-down หากสถานะล็อกอยู่ (`isBoxUnlocked === false`) ให้ถมเลเยอร์ภาพไอคอนแม่กุญแจสีเทาทับซ้อนไว้ด้านบนและปิดกั้นสิทธิ์การคลิก (pointer-events-none) แต่ถ้าเงื่อนไขครบถ้วน ให้สลับภาพไอคอนแม่กุญแจเป็นสีทองอร่าม พร้อมเปิดสิทธิ์ให้สามารถกดหรือโต้ตอบได้ทันที
  * โครงสร้างหน้าต่างป๊อปอัปควิซ (Quiz Popup Layout): เมื่อเปิดหน้าต่างควิซ ให้สลัดกล่องคำถามจัดเรียงชุดตัวเลือกรูปภาพสี่เหลี่ยมสีน้ำทั้ง 4 ช้อยส์ให้อยู่ในรูปแบบระบบตารางสี่ช่องสมดุล (Grid Grid-cols-2 Gap-3) ย่อขนาดพอดีกับสัดส่วนหน้าจอมือถืออัตโนมัติเพื่อให้แฟนจิ้มเลือกภาพได้สะดวก

3. Animation & Grand Finale Interaction (ระบบแอนิเมชันไขกุญแจและฉากจบระเบิดรูปถ่าย)
- เครื่องมือที่ใช้: Framer Motion และระบบการลากอิสระ (Drag Sandbox)
- ตรรกะการทำงาน:
  * แอนิเมชันเปิดสิทธิ์กล่อง: ทันทีที่เงื่อนไขด่านครบถ้วนและแม่กุญแจเปลี่ยนเป็นสีทอง ให้ระบบสั่งเรียกประมวลผลไฟล์เสียงเอฟเฟกต์ปลดล็อก (`unlock.mp3`) ควบคู่กับการเล่นแอนิเมชันไอคอนแม่กุญแจสั่นและลอยหายไป
  * เฟสไขกุญแจรับรางวัล (Key Drag & Unlock): เมื่อเคลียร์ควิซภาพ 2 ข้อจบลง ป๊อปอัปจะปิดตัว และระบบจะสั่งรันแอนิเมชันภาพลูกกุญแจสีน้ำวินเทจ (`key.png`) ให้ลอยฟึ่บขึ้นมาวางเด่นข้างตัวกล่องบนโต๊ะหลัก โดยเปิดฟังก์ชันคำสั่งลาก (drag) บนตัวกุญแจนี้ เพื่อให้แฟนสามารถลากลูกกุญแจเคลื่อนที่ไปไขสัมผัสโดนขอบเขตตัวกล่องของขวัญ
  * เอฟเฟกต์ฉากจบระเบิดความทรงจำ (Grand Finale Explosion): ทันทีที่พิกัดกุญแจสัมผัสโดนกล่อง (Collision Detected) สั่งให้ภาพฝากล่อง (`pha.png`) แยกตัวลอยกระจายออกจากตัวกล่องหลักทันที พร้อมทั้งสั่งดึงข้อมูลรูปภาพคู่ทั้ง 2 ใบที่แฟนเลือกไว้จากตัวแปร `selectedQuizImages` พุ่งกระเด้งกระจายตัว (Explode Animation) ลอยกระจัดกระจายออกมาจากใจกลางกล่องขึ้นมาถมเต็มหน้าจอร่วมกับไอเทมชิ้นส่วนความทรงจำอื่นๆ
  * ระบบกระดานลากเกลี่ยรูปภาพอิสระ (Drag Sandbox Area): รูปถ่ายและไอเทมคู่รักทั้งหมดที่พุ่งระเบิดกระจายตัวออกมาค้างอยู่บนหน้าจอตอนฉากจบ จะถูกเปิดคำสั่งควบคุมคุณสมบัติการลากเคลื่อนย้ายอิสระแบบไร้ขอบเขต (`<motion.div drag dragConstraints={{ left: 0, right: 0, top: 0, bottom: 0 }} />` หรือปล่อยอิสระตามขนาดจอ) แฟนสามารถใช้นิ้วจิ้มเกลี่ย สะบัด ปัดขยับจัดระเบียบตำแหน่งรูปภาพคู่บนหน้าจอเล่นได้อย่างอิสระตามใจชอบในเฟสสุดท้ายนี้
