### 🍳 Step 4_1: New Tortilla Mini-Game Specs & Folding Logic (ระบบมินิเกมตอติญ่าโฉมใหม่และการพับแป้ง 2 จังหวะ)

1. State & Phase Logic Flow (ระบบจัดการสถานะและการควบคุมเฟสทั้ง 4)
- เครื่องมือที่ใช้: React State (useState) และ LocalStorage
- ตรรกะการทำงาน:
  * Phase 1 (Chopping): ตั้งตัวแปรสเตตสะสมยอดการคลิก (Chop Counter) เริ่มต้นที่ 0 ควบคุมรูปภาพไก่ดิบ /images/chicken.png เมื่อผู้ใช้กดคลิก/แตะครบ 6 ครั้งเต็ม จะทำการ Unmount หน้าจอนี้ และส่งต่อไปยัง Phase 2 ทันทีอย่างอัตโนมัติ โดยในระหว่างกดให้มีเอฟเฟกต์การสั่นไหวสั้นๆ (Scale/Rotate) เพื่อแสดงการตอบสนอง
  * Phase 2 (Cooking Loading Bar): หน้าจอจำลองการทำอาหาร โดยสร้างตัวแปรหลอดสถานะ (Loading Bar) ด้วย Tailwind ค่อย ๆ วิ่งเพิ่มความกว้างจาก 0% ไปถึง 100% ล็อกเวลาหน่วงคงที่ 6 วินาทีเต็ม (duration: 6000ms) มีข้อความ Text ระบุว่า "Cooking..." วิ่งหรือกะพริบอยู่เหนือหลอดโหลด เมื่อหลอดเต็ม 100% ให้ทำการข้ามเข้าสู่ Phase 3 ทันที
  * Phase 3 (Assembly): ระบบจัดวางวัตถุดิบบนแผ่นแป้งตอติญ่าขนาดใหญ่รูปภาพ /images/tortilla.png ทำหน้าที่เป็นพื้นที่ Drop Zone หลักตรงกลางจอ
    - ชิ้นส่วนผักกาด (/images/lettuce.png) และไก่สุก (/images/chicken_2.png) เปิดระบบลากอิสระ (Framer Motion drag) เพื่อนำไปวางไว้กึ่งกลางแผ่นแป้ง
    - ลอจิกพิเศษขวดซอส (Sauce Click Trigger): ตัวขวดซอส /images/sauce.png ด้านนอก ห้ามใช้ระบบลาก ล็อกเงื่อนไขให้ดักจับเหตุการณ์คลิก (onClick) เท่านั้น ทันทีที่ผู้ใช้กดคลิกที่ขวดซอส ให้สั่งซ่อนรูปขวดซอสเดิมทิ้งไป (Set visibility เป็น false) และเรนเดอร์รูปซอสราดสำเร็จ /images/sauce_2.png โผล่ซ้อนบนแผ่นแป้งตรงกลางแทนทันที
    - เมื่อผู้ใช้จัดวางวัตถุดิบค้างบนแป้งครบทั้ง 3 ชิ้น (ผัก, ไก่, ซอส_2) ระบบจะเปิดสวิตช์แสดงปุ่มเด่นชัดพิมพ์ว่า [ พับแป้ง ] เพื่อสั่ง Warp เข้าสู่ Phase สุดท้าย
  * Phase 4 (2-Step Folding): ยกระดับเลเยอร์และขยายขนาดแผ่นตอติญ่าให้ออกมามีขนาดใหญ่ขึ้นกว่าด่านก่อนหน้านี้อย่างเห็นได้ชัดเพื่อเตรียมรับคำสั่งพับแป้ง โดยล็อกระบบพับแป้งด้วยการคลิกแยกฝั่งแบบ 2 จังหวะถ้วน (ซ้าย และ ขวา):
    - เมื่อผู้ใช้คลิกที่พื้นที่ฝั่งซ้ายของจอภาพ/แผ่นแป้ง: ให้ทำการเปลี่ยนสเตตรูปภาพแผ่นแป้งอัปเดตเป็น /images/Left.png
    - เมื่อผู้ใช้คลิกที่พื้นที่ฝั่งขวาของจอภาพ/แผ่นแป้ง: ให้ทำการเปลี่ยนสเตตรูปภาพแผ่นแป้งอัปเดตเป็น /images/right.png
    - การเปลี่ยนรูปภาพทั้งสองจังหวะ ให้ห่อหุ้มด้วย <motion.img> ของ Framer Motion โดยใช้เอฟเฟกต์แอนิเมชันสลับรูปภาพแบบสมูทนุ่มนวล (เช่น ใส่ initial={{ opacity: 0, scale: 0.95 }} animate={{ opacity: 1, scale: 1 }} transition={{ duration: 0.3 }}) ห้ามให้รูปภาพตัดเปลี่ยนแบบแข็งกระด้างเด็ดขาด
    - เมื่อระบบตรวจพบการคลิกพับแป้งครบทั้ง 2 ฝั่งสำเร็จ ให้ขึ้นข้อความแสดงคำว่า "พับเรียบร้อย" ตรงกลางหน้าจอ พร้อมเปลี่ยนรูปคอมโพเนนต์ทั้งหมดกลายเป็นรูปแป้งห่อเสร็จสมบูรณ์จากไฟล์ /images/tortilla_complete.png ทันที
    - ลอจิกส่งท้ายด่าน: ให้ระบบค้างภาพความสำเร็จ tortilla_complete.png นี้ไว้เป็นเวลา 2 วินาทีเต็ม จากนั้นสั่งบันทึกสถานะผ่าน localStorage.setItem('isTortillaCooked', 'true') เพื่อยุติมินิเกมและตัดสลับหน้าจอเข้าสู่ FinalScene (ฉากเปิดกล่องระเบิดไอเทมกระจายรอบนอก) ทันที

2. Asset & UI Layout Design (การจัดวางโครงสร้างทรัพยากรและเลเยอร์หน้าจอ)
- เครื่องมือที่ใช้: Tailwind CSS 100% (Flexbox & Grid Layout)
- ตรรกะการทำงาน:
  * พิกัดเรียกใช้ไฟล์ทรัพยากรรูปภาพทั้งหมดอ้างอิงจากโฟลเดอร์ public/images/ ประกอบด้วย: chicken.png, tortilla.png, lettuce.png, chicken_2.png, sauce.png, sauce_2.png, Left.png, right.png, และ tortilla_complete.png
  * การจัดเลเยอร์พื้นที่ด่านวางไส้และด่านพับแป้ง: ใช้การคุมตำแหน่งสัมพัทธ์ (relative) และแบ่งพื้นที่กึ่งกลางจอภาพให้เคลียร์ โดยกำหนดค่าความสูงของลำดับชั้น (Z-Index) วัตถุดิบและเนื้อสัตว์ให้อยู่ระดับต่ำกว่าแผ่นแป้งปีกพับ เพื่อให้เมื่อเกิดเหตุการณ์คลิกเปลี่ยนสลับรูปภาพพับใน Phase 4 เลเยอร์ของแผ่นแป้งที่แสดงผลขึ้นมาใหม่จะถมทับปิดบังวัตถุดิบไส้ด้านในได้อย่างมิดชิดร้อยเปอร์เซ็นต์

3. Animation & Interaction Mapping (ระบบควบคุมการเคลื่อนไหวและเอฟเฟกต์)
- เครื่องมือที่ใช้: Framer Motion
- ตรรกะการทำงาน:
  * ใน Phase 1 (หั่นวัตถุดิบ): ใช้คำสั่ง whileTap ของ Framer Motion เพื่อลดขนาด scale ลงมาเหลือ 0.95 ทุกครั้งที่กดสับไก่ดิบ
  * ใน Phase 3 (ระบบลากวางวัตถุดิบ): ตั้งค่าแรงดีดกลับของวัตถุดิบ (dragTransition) ด้วยค่าความแข็งของสปริงสูง (bounce: 0) เพื่อให้วัตถุดิบดีดกลับสล็อตเริ่มต้นทันทีหากผู้ใช้ปล่อยมืออยู่นอกพื้นที่แผ่นแป้งหลัก และสั่งปิดระบบลาก (ล็อกตำแหน่งตายตัว) ทันทีที่ลากเข้าสู่พื้นที่แผ่นแป้งตอติญ่าสำเร็จ
  * ในการเปลี่ยนผ่านเฟส (Phase Transition): ใช้คอมโพเนนต์ AnimatePresence ควบคุมเอฟเฟกต์การ Fade-in และ Fade-out ระหว่างฉากหั่น, ฉากหลอดโหลด, ฉากวางไส้ และฉากพับแป้ง เพื่อความต่อเนื่องและลื่นไหลของหน้าจอ โดยไม่มีการตัดภาพแข็งกระด้าง
