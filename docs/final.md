### 💖 Step 5: Grand Finale & Interactive Sandbox Workflow (กระบวนการทำงานฉากจบและระบบปุ่มแกง)

1. State & Interaction Logic Flow (ระบบจัดการสถานะและตรรกะควบคุมพิธีเปิดกล่อง)
- เครื่องมือที่ใช้: React State (useState) และ Event Listener ของระบบ Drag/Drop
- ตรรกะการทำงาน:
  * ตรรกะตรวจจับการไขกุญแจ (Key Drop Zone): ดักจับพิกัดการลากลูกกุญแจ (`key.png`) เมื่อปล่อยวัตถุลงบนพื้นที่กล่องของขวัญ (`box.png`) ระบบจะสลับค่าสถานะล็อกกุญแจเป็นสำเร็จ (isKeyUnlocked: true) 
  * ระบบเปิดฝากล่องด้วยมือ (Manual Lid Opening State): เมื่อไขกุญแจแล้ว ระบบจะเปิดสิทธิ์ให้สามารถใช้นิ้วหรือเมาส์คลิกลากแผ่นชิ้นส่วนฝากล่อง (`pha.png`) ได้จริงในแนวดิ่ง/แนวนอน โดยตั้งเงื่อนไขเช็กระยะ (Threshold) หากแฟนลากฝากล่องเลื่อนออกห่างจากตัวกล่องท่อนล่างเกินสัดส่วนพิกัดที่กำหนด (เช่น ลากพ้นขอบเกิน 100px) ระบบจะปรับสถานะฝากล่องเป็นหายไป (isLidRemoved: true) แล้วสั่งรันเอฟเฟกต์เฟดจางหาย (Fade Out) และสั่งเริ่มเฟสระเบิดความทรงจำทันที
  * ตรรกะการดึงข้อมูลระเบิดไอเทม (Memory Burst Item Collector): ระบบจะทำการรวบรวมชิ้นส่วน Asset จากสเต็ปก่อนหน้าทั้งหมดมารวมกันในอาณาเขตฉากจบ ประกอบด้วย:
    - แผ่นแป้งตอติญ่าม้วนสำเร็จจากมินิเกม Step 4_1 (ดึงมาเมื่อค่า `isTortillaCooked === true`)
    - รูปถ่ายคู่สไตล์โพลารอยด์ทั้งหมด 20 ใบจากแฟ้มไดอารี่ Step 4_3 (`photo-01.png` ถึง `photo-20.png`)
    - รูปภาพช้อยส์คำตอบควิซจำนวน 2 ใบที่แฟนเลือกจิ้มไว้ใน State `selectedQuizImages` จากด่าน Step 4_4
  * สถานะจดหมายลับและปุ่มกด (Secret Letter & Dynamic Button State): 
    - ใช้ Boolean State ควบคุมการเปิดแสดงผลการ์ดจดหมายลับ (`isLetterOpen: false`) ซึ่งจะถูกเปลี่ยนเป็น true เมื่อแฟนกดคลิกที่ไอคอนซองจดหมาย (`envelope.png`) ที่ก้นกล่องหลังจบเอฟเฟกต์ระเบิด
    - ใช้ State พิกัดวัตถุตำแหน่งแกน X และ Y (เช่น noBtnPosition: {x: 0, y: 0}) สำหรับควบคุมการสุ่มย้ายที่ของปุ่ม [ ไม่เป็น ]
    - ใช้ State เก็บตัวเลขเวลา (Timer State) นับจาก 0 ถึง 60 วินาทีทันทีที่จดหมายเปิดออก เพื่อนำไปคำนวณสเกลขนาดความใหญ่ของปุ่ม [ เป็น ]

2. Asset & UI Layout Design (การจัดวางโครงสร้างเลเยอร์ Sandbox และกล่องจดหมาย)
- เครื่องมือที่ใช้: Tailwind CSS 100%
- ตรรกะการทำงาน:
  * โครงสร้างพื้นที่แซนด์บ็อกซ์ (Interactive Sandbox Container): กำหนดขอบเขตเต็มหน้าจอแสดงผล (w-full h-screen overflow-hidden relative) เพื่อทำหน้าที่เป็นกล่องกั้นอาณาเขตไม่ให้วัตถุความทรงจำที่ลอยกระจายหลุดทะลุออกไปนอกขอบจอคอมพิวเตอร์หรือมือถือ
  * ลำดับเลเยอร์ (Z-Index Hierarchy): จัดวางตำแหน่งวัตถุให้ถูกต้อง (กล่องท่อนล่างอยู่ชั้นล่าง -> ชิ้นส่วนฝากล่องและกุญแจอยู่ชั้นบน -> ซองจดหมายลับซ่อนอยู่ด้านในสุดของตัวกล่องท่อนล่าง)
  * โครงสร้างป๊อปอัปจดหมายลับ (Secret Letter Layout): จัดวางกล่องการ์ดจดหมายไว้กึ่งกลางหน้าจอแบบสัมบูรณ์ (Absolute Center) ถมสีพื้นหลังกระดาษสีน้ำนุ่มนวล จัดแบ่งสล็อตข้อความระบบพิมพ์ดีดไว้ส่วนบน และจัดวางแผงควบคุมปุ่ม 2 ปุ่มไว้ส่วนล่างสุดของการ์ด

3. Animation & Interactive Sandbox Mapping (ระบบควบคุมการเคลื่อนไหว แซนด์บ็อกซ์ และปุ่มแกง)
- เครื่องมือที่ใช้: Framer Motion และ Canvas Confetti
- ตรรกะการทำงาน:
  * แอนิเมชันกระจายตัว (Explode Effect): ทันทีที่ฝากล่องถูกเปิดออก สั่งให้ไอเทมความทรงจำทุกชิ้น (แผ่นแป้ง, รูปถ่าย 20 ใบ, รูปควิซ 2 ใบ) เล่นอนิเมชันพุ่งกระเด็นออกจากพิกัดกึ่งกลางกล่องกระจายออกไปรอบทิศทางแบบสุ่มค่าพิกัด (Randomize ค่า X, Y และองศาการหมุน Rotate) ด้วยน้ำหนักแรงดีดที่นุ่มนวล
  * ระบบกระดานลากเกลี่ยรูปภาพอิสระ (Drag Sandbox Area): ไอเทมความทรงจำทุกชิ้นที่กระจายอยู่บนหน้าจอ ต้องเปิดใช้งานคุณสมบัติคำสั่งลากคลุม (`drag`) ของ Framer Motion แฟนสามารถใช้นิ้วจิ้มเกลี่ย สะบัด ปัดกองรูป หรือจัดระเบียบตำแหน่งรูปถ่ายคู่บนหน้าจอเล่นได้อย่างอิสระตลอดเวลาโดยไม่มีระบบ constraints ล็อกพิกัดกลับ
  * เอฟเฟกต์พิมพ์ดีดจดหมาย (Typewriter Effect): เมื่อกดซองจดหมาย การ์ดจะเล่นอนิเมชันเด้งขยายใหญ่ขึ้นมา (Scale-in) จากนั้นตัวอักษรข้อความ "เป็นแฟนกันได้มั้ย" จะค่อยๆ เล่นอนิเมชันแสดงผลโผล่ขึ้นมาทีละตัวอักษรจนครบประโยค
  * ตรรกะระบบปุ่ม [ ไม่เป็น ] (ปุ่มโทรลสุ่มวาร์ป):
    - ผูก Event ล่อเป้าไว้ที่ตัวปุ่มผ่านคำสั่ง `onMouseEnter` (บน Desktop) และ `onTouchStart` (บนมือถือ)
    - ทันทีที่แฟนขยับเมาส์ไปชี้หรือใช้นิ้วพยายามจะจิ้มโดน ให้สั่งทำงานคลาสแอนิเมชันสั่นสะเทือนหน้าจอหลัก (Screen Shake Animation) 1 จังหวะ
    - พร้อมทั้งสั่งรันฟังก์ชัน Math.random() สุ่มคำนวณพิกัดแกน X และ Y ชุดใหม่ภายในกรอบหน้าจอ แล้วสั่งให้ตัวปุ่ม [ ไม่เป็น ] วาร์ปสลับย้ายตำแหน่งหนีหายไปอยู่พิกัดใหม่ทันทีในเสี้ยววินาที ทำให้กดใช้งานไม่ได้จริง
  * ตรรกะระบบปุ่ม [ เป็น ] (ปุ่มล็อกมงขยายร่าง):
    - ตั้งค่าฟังก์ชันเพิ่มขนาดอัตโนมัติ (Interpolation Scaling): ผูกขนาดปุ่มเข้ากับค่าตัวนับเวลา (Timer) ทุกๆ วินาทีที่ผ่านไป ปุ่ม [ เป็น ] จะค่อยๆ ขยายมาตรส่วนขนาดของตัวเองขึ้นอย่างช้าๆ (เช่น scale: 1 + (time * 0.03)) ไปสิ้นสุดขนาดใหญ่ยักษ์ที่สุดเมื่อเวลาดำเนินไปครบ 1 นาที (60 วินาที) เพื่อกดดันและดึงดูดสายตาให้แฟนกดปุ่มนี้ให้ได้
  * เฟสฉลองความสำเร็จ (Grand Finale Success): ทันทีที่แฟนกดคลิกปุ่ม [ เป็น ] ได้สำเร็จ ให้ระบบสั่งปิดหน้าต่างจดหมายลง พร้อมทำการเรียกใช้งานไลบรารียิงพลุพาร์ทิเคิลกระดาษสีแนวรูปหัวใจสลับสีพาสเทลระเบิดฟู่ออกมาพุ่งกระจายเต็มหน้าจอแบบชุดใหญ่ต่อเนื่อง (Continuous Heart Canvas Confetti) เพื่อเฉลิมฉลองการตกลงเป็นแฟนกันอย่างสมบูรณ์แบบ
