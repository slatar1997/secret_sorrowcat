### 🎬 Step 3: Cinematic Sliding View (ระบบอนิเมชันรถไฟวิ่งผ่านหน้าต่าง)

1. State & Phase Logic Flow (ระบบจัดการสถานะและตรรกะตัวแปร)
- เครื่องมือที่ใช้: React State และ Callback ของระบบอนิเมชัน (onAnimationComplete)
- ตรรกะการทำงาน:
  * เปิดสิทธิ์การเข้าฉากนี้เมื่อระบบดักจับได้ว่า Step 2 ทำการประมวลผลตั๋วเสร็จสิ้น (เข้าสู่สเต็ปกระบวนการเคลื่อนที่) โดยถมพื้นหลังคงที่เป็นภาพภายในโบกี้รถไฟ (Train Cabin Background)
  * หลีกเลี่ยงการใช้ระบบตั้งเวลาจากฟังก์ชันภายนอก (เช่น setTimeout วินาทีตายตัว) ในการตัดฉาก เพื่อป้องกันปัญหาแอนิเมชันเคลื่อนภาพยังไม่จบแต่ฉากถูกตัดก่อน โดยระบบจะใช้ฟังก์ชันดักจับแอนิเมชันสิ้นสุดลงจริง (onAnimationComplete) เป็นตัวส่งสัญญาณคำสั่ง (Trigger) เคลื่อนภาพเข้าสู่ด่านถัดไป
  * ตรรกะการเปลี่ยนฉากปลายทาง (Transition Out): ทันทีที่ระบบได้รับสัญญาณจบแอนิเมชัน ให้เปิดสวิตช์ทำงานเอฟเฟกต์ผสมผสาน 3 รูปแบบพร้อมกัน คือ การลดค่าความโปร่งแสงให้เลือนหาย (Fade) + การปรับความเบลอของภาพ (Blur) + แสงวาบสีขาวนวลความสว่างสูงช่วงสั้นๆ (White Flash) ผ่านการควบคุมค่า Opacity และระบบตัวกรอง Filter เพื่อเปลี่ยนผ่านเข้าสู่ฉากโต๊ะทำงานหลัก (Desk Scene) อย่างสมูทที่สุด

2. Asset & UI Layout Design (การจัดวางโครงสร้างและเลเยอร์หน้าจอ)
- เครื่องมือที่ใช้: Tailwind CSS 100%
- ตรรกะการทำงาน:
  * การจัดโครงสร้างเลเยอร์หน้ากาก (Static Mask Layer): วางโครงสร้าง "กรอบหน้าต่างรถไฟ" ไว้ตรงกลางจอภาพเป็นองค์ประกอบนิ่งสนิท ไม่มีการเคลื่อนไหวใดๆ 100% โดยสั่งเปิดใช้งานคลาสปิดบังขอบวัตถุส่วนเกิน (overflow-hidden) บนตัวกรอบหน้าต่าง เพื่อทำหน้าที่เป็นตัวกรองกล่องบังตา (Mask) ซ่อนชิ้นส่วนภาพวิวทั้งหมดที่ล้นออกไปนอกอาณาเขตหน้าต่าง
  * การจัดโครงสร้างเลเยอร์ภาพเคลื่อนไหว (Moving Container Layer): สร้างกล่องแถวเรียงกระดานแนวนอนยาว (Flexbox Container) ซ่อนไว้ตำแหน่งเลเยอร์ด้านหลังของกรอบหน้าต่างรถไฟ ภายในกล่องทำการจัดเรียงไฟล์ภาพวิวสีน้ำสไตล์ลอนดอน 3 ภาพต่อแถวกันตามลำดับโครงสร้างเนื้อเรื่อง:
    * ภาพที่ 1: london-1.jpg (หอนาฬิกา Big Ben)
    * ภาพที่ 2: london-2.jpg (ทัศนียภาพแม่น้ำเทมส์)
    * ภาพที่ 3: london-3.png (สะพานทาวเวอร์บริดจ์ Tower Bridge)

3. Animation & Interaction Mapping (ระบบควบคุมการเคลื่อนไหวและเอฟเฟกต์)
- เครื่องมือที่ใช้: Framer Motion
- ตรรกะการทำงาน:
  * กลไกการเคลื่อนที่ภาพ: สั่งแอนิเมชันปรับแต่งค่าพิกัดไปที่ตัวกล่องแถว Container แถบภาพวิวด้านหลังเท่านั้น โดยเลื่อนตำแหน่งระยะเยื้องแนวนอนจากศูนย์ไปสู่พิกัดค่าติดลบทางฝั่งซ้าย (transform: translateX จากพิกัดเริ่มต้น 0 ไปยังค่าติดลบปลายทาง) เพื่อสร้างเอฟเฟกต์แถวภาพไหลผ่านช่องหน้าต่างจากขวาไปซ้าย
  * **ข้อห้ามเชิงเทคนิค:** ห้ามใช้คำสั่งขยับโครงสร้าง Layout เช่น การปรับค่าระยะขอบ (Margin) หรือปรับพิกัดตำแหน่งสัมพัทธ์ (Position Absolute: Left/Right) ในการทำแอนิเมชันเด็ดขาด เพื่อเปิดสิทธิ์ให้ตัวบราวเซอร์เรียกใช้การประมวลผลผ่านการเร่งความเร็วฮาร์ดแวร์ (Hardware Acceleration) ทำให้ภาพเคลื่อนไหวได้ลื่นไหลระตับสูงสุด (Smooth Motion) บนหน้าจอมือถือ
  * การผ่อนน้ำหนักและความเร็ว (Easing & Duration): กำหนดค่ารูปแบบการผ่อนน้ำหนักเคลื่อนภาพเป็นแบบเส้นตรงความเร็วสม่ำเสมอเท่ากันตลอดสาย (ease: "linear") เพื่อจำลองสปีดความเร็วของรถไฟที่กำลังแล่นอยู่จริง และตั้งค่าระยะเวลายาวเป็นพิเศษ (ประมาณ 10–20 วินาที) เพื่อขับเน้นอารมณ์ความละมุนสไตล์ภาพยนตร์ (Cinematic Feel)
  * ตรรกะจังหวะหยุดภาพปลายทาง: เมื่อระบบเคลื่อนภาพไหลมาจนสุดแถวที่พิกัดภาพใบสุดท้าย (london-3.png สะพาน Tower Bridge) ให้ตัวภาพหยุดนิ่งตรึงล็อกอยู่กลางช่องหน้าต่างรถไฟ และตั้งเงื่อนไขให้ระบบชะลอค้างหน้าจอเอาไว้เป็นเวลา 0.5–1 วินาที เพื่อให้แฟนได้ซึมซับภาพสถานีปลายทางให้ชัดเจน ก่อนที่ตัวระบบจะส่งสัญญาณ Callback ดำเนินการรันฟังก์ชันข้ามฉากตามตรรกะข้อที่ 1
