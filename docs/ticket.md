### 🎫 Step 2: Dynamic Ticket System & UI Layout Specs (ระบบตั๋วแปรผันตามชื่อและดีไซน์)

1. State & Logic Flow (ระบบจัดการสถานะและตรรกะตัวแปร)
- เครื่องมือที่ใช้: React State (useState)
- ตรรกะการทำงาน:
  * ใช้ State ชื่อ passengerName สำหรับผูกเข้ากับช่อง Input บนหน้าจอเพื่อดักจับและเก็บค่าตัวอักษรชื่อที่แฟนพิมพ์เข้ามา
  * เมื่อมีการกดปุ่ม "ออกตั๋ว" ระบบจะประมวลผลและส่งต่อค่า passengerName นี้เข้าสู่ Component การ์ดตั๋วรถไฟลอนดอนโดยอัตโนมัติ เพื่อนำไปแสดงผลบนหน้าตั๋วในเฟสถัดไป
  * ใช้ State ประเภท Boolean (เช่น isFlipped) คอยดักจับอีเวนต์การคลิก (onClick) ที่ตัวแผ่นตั๋วรถไฟ เพื่อสลับค่าสถานะในการสั่งพลิกสลับหน้าการ์ดไปมา

2. Ticket UI Design & Layout (การจัดวางโครงสร้างและเลเยอร์หน้าจอด้วย Tailwind CSS)
- เครื่องมือที่ใช้: Tailwind CSS 100%
- ตรรกะการทำงาน:
  * ข้อกำหนดโทนสี: ถมเนื้อกระดาษด้วยสีครีมเก่าสไตล์วินเทจ (bg-[#FDFBF7]) และใช้สีส้มชานมพาสเทลอบอุ่น (border-[#FFA07A]) ในการตัดเย็บเส้นขอบ เพื่อจำลองมู๊ดรถไฟใต้ดินลอนดอนแต่ให้ความรู้สึกซอฟต์หวาน
  * โครงสร้างสัดส่วนตั๋ว: ออกแบบกรอบสี่เหลี่ยมผืนผ้าแนวนอน ใช้คลาสขีดเส้นประ (border-dashed) สีน้ำตาลเพื่อแบ่งพื้นที่ตั๋วออกเป็น 2 ส่วนเด็ดขาด คือ ส่วนหลักฝั่งซ้าย (Main Area) ขนาดพื้นที่ 70% และส่วนตอฉีกฝั่งขวา (Stub Area) ขนาดพื้นที่ 30%
  * การเจาะรูตั๋ว: จัดวางกล่อง Div วงกลม (rounded-full) ซ่อนพิกัดขอบไว้แบบ absolute ทั้งฝั่งซ้ายและขวา เพื่อสร้างรอยบากเว้าลึกครึ่งวงกลมให้เหมือนโครงสร้างตั๋วรถไฟกระดาษดั้งเดิม
  * เลเยอร์สามมิติ (3D Setup): กำหนดค่ามิติความลึก (Perspective) บนกล่องควบคุมหลัก และเปิดคำสั่งซ่อนพื้นผิวด้านหลัง (backface-hidden) เพื่อให้ตัวการ์ดตั๋วลอยเด่นขึ้นมาจากพื้นหลัง และพร้อมรับคำสั่งพลิกการ์ดสลับหน้า-หลังโดยไม่เกิดอาการภาพซ้อนทะลุกรอบ

3. Ticket Content Mapping (การจัดวางข้อมูลเนื้อหาบนตั๋วและหลังการ์ด)
- เครื่องมือที่ใช้: Tailwind CSS ย่อยร่วมกับสถาปัตยกรรม Font ของระบบ
- ตรรกะการทำงาน:

* ฝั่งพื้นที่ตั๋วหลักด้านซ้าย (Main Ticket Area - 70%):
  - ส่วนหัว (Header): วาดสัญลักษณ์โลโก้รถไฟใต้ดินทรงกลมด้วยคำสั่ง Div และพิมพ์ข้อความกำกับ "LONDON TRANSIT" โดยใช้ฟอนต์หัวข้อตัวหนามน (Font: Itim)
  - ส่วนตารางข้อมูล (Data Grid): จัดระเบียบเนื้อหาแบ่งเป็น 2 คอลัมน์อย่างสมดุล ประกอบด้วย:
    * Name: ดึงค่าตัวแปรแปรผันมาแสดงผลตรงนี้ `[passengerName]` โดยแสดงผลเป็นตัวหนาสีน้ำตาลเข้ม คุมสไตล์ด้วยฟอนต์ลายมือ (Font: Mali)
    * Destination: พิมพ์ข้อความล็อกค่าตายตัวคำว่า "Together"
    * Start Date: พิมพ์ข้อความล็อกค่าวันเดินทางคำว่า "19 June 2026"
    * Valid Until: พิมพ์ข้อความสัญญารักล็อกค่าคำว่า "No expired"
  - ส่วนท้าย (Footer): ใส่ข้อความคำว่า "Love Express" ปรับค่าความโปร่งแสงให้จางลง (Opacity ต่ำ) เพื่อทำหน้าที่เป็นสัญลักษณ์ลายน้ำพื้นหลัง

* ฝั่งพื้นที่ตอนฉีกตั๋วด้านขวา (Stub Area - 30%):
  - พิมพ์ข้อความระดับชั้น Class: "1ST" โดยตบแต่งด้วยการครอบกล่องสี่เหลี่ยมมนสีส้มและใช้ตัวอักษรสีขาวหนาเด่นชัด
  - พิมพ์ข้อความระบุตำแหน่งที่นั่ง Seat: "Heart" / ระบุเวลา Time: "20.26" / ระบุรหัสหมายเลขตั๋ว Number: "190626"
  - ส่วนบาร์โค้ดจำลอง (Barcode): จัดสลิตเรียงกล่อง Div เส้นตรงสีดำแนวดิ่งเรียงชิดติดกันถี่ๆ โดยสลับการตั้งค่าความกว้างโค้ดหนา-บางไม่เท่ากัน (เช่น w-0.5, w-1, w-1.5) ที่บริเวณขอบล่างสุด เพื่อจำลองแท่งบาร์โค้ดสินค้าโดยไม่ต้องพึ่งพาไฟล์รูปภาพภาพนอก

* ฝั่งพื้นที่ด้านหลังการ์ดตั๋ว (Ticket Backside):
  - เมื่อเกิดสถานะพลิกกลับแกน Y 180 องศา ให้ถมเนื้อสีครีมกระดาษเก่าแบบเดียวกัน และแสดงผลข้อความตรงกลางหน้าจอระบุประโยคว่า "ticket for love" คุมสไตล์ด้วยฟอนต์ลายมือ

4. Audio & Presentation Mapping (ระบบคุมความเคลื่อนไหวและสัญญาณเสียง)
- เครื่องมือที่ใช้: JavaScript Audio API และ Framer Motion
- ตรรกะการทำงาน:
  * ทันทีที่ผู้ใช้สั่งคลิกยืนยันข้อความออกตั๋ว ระบบจะสั่งเล่นอนิเมชันนำการ์ดตั๋วเลื่อนไถลแกน Y สไลด์จากใต้ขอบล่างของหน้าจอโผล่ขึ้นมาหยุดตรงตำแหน่งกึ่งกลาง พร้อมสั่งเรียกประมวลผลไฟล์เสียงเครื่องพิมพ์ตั๋วทำงาน (ticket-print.mp3) ควบคู่ไปในเวลาเดียวกัน
  * เมื่อแฟนทำการคลิกเมาส์หรือจิ้มนิ้วลงบนเนื้อการ์ด ให้สั่งทำงานแอนิเมชันเคลื่อนไหวหมุนเปลี่ยนพิกัดแกนนอน (animate={{ rotateY: isFlipped ? 180 : 0 }}) เพื่อจำลองเอฟเฟกต์การพลิกแผ่นตั๋วสามมิติแบบนุ่มนวล
