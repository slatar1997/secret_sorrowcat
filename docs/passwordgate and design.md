## 🎨 Design Theme & Aesthetic (ข้อกำหนดธีมการออกแบบและงานภาพรวม)

1. Art Style & Visual Direction
- รูปแบบงานภาพ (Art Style): กำหนดทิศทางเป็นแนวสีน้ำและภาพวาดลายเส้นมือ (Watercolor & Hand-drawn) เน้นมู๊ดโฮมเมด อบอุ่น นุ่มนวล ชิ้นส่วนกราฟิกทุกชิ้นต้องคุมขอบให้มีความฟุ้งและไม่แข็งกระด้าง
- ระบบการตอบสนองหน้าจอ (Responsive Layout): โครงสร้างหน้าเว็บทั้งหมดต้องรองรับสัดส่วนการแสดงผลแบบ Mobile-First โดยเฉพาะบนบราวเซอร์มือถือ (Safari และ Chrome) และขยายขนาดรับกับหน้าจอ Desktop ได้โดยที่สัดส่วนไม่บิดเบี้ยว

2. Color Palette Configuration (โทนสีพาสเทลธรรมชาติ)
- สีพื้นหลังหน้าจอปลดล็อกและกล่องของขวัญ: ถมสีกระดาษบลัชออนพาสเทลจาง (สีหลักโทน #FFF0F5) เพื่อสร้างอารมณ์แรกพบที่หวานละมุน
- สีพื้นหลังโต๊ะทำงานและเครื่องเล่นไวนิลหลัก: ใช้โทนสีน้ำตาลเนื้อไม้อ่อน (Warm Wood Tone) เพื่อขับเน้นอารมณ์ความคลาสสิกและทำให้วัตถุบนโต๊ะดูลอยมีมิติ

3. Typography & Fonts Integration (การจัดการระบบตัวอักษรผ่าน Google Fonts)
- หัวข้อหลักและข้อความตัวโต (Headings & Titles): เรียกใช้งานฟอนต์ "Itim" ควบคุมลักษณะตัวหนา ขอบมน ดูมีความขี้เล่น สนุกสนาน และเตะตาเด่นชัด
- เนื้อหาทั่วไปและซับไตเติล (Body Text & Subtitles): เรียกใช้งานฟอนต์ "Mali" จำลองลักษณะลายมือน่ารัก เรียบร้อย อบอุ่น รองรับการแสดงผลจัดระเบียบคำได้ถูกต้องสมบูรณ์ทั้งภาษาไทยและภาษาอังกฤษ

## 🎮 Game Flow & Features (ลำดับและลูกเล่นในเกม)

### 🔐 Step 1: Password Gate Specs (ระบบรหัสผ่าน "2003" และดีไซน์ปุ่มพาสเทล)

1. State & Logic Flow (ระบบจัดการสถานะและตรรกะตัวแปร)
- เครื่องมือที่ใช้: React State (useState) และ Event Listener ของหน้าต่างบราวเซอร์
- ตรรกะการทำงาน:
  * ตั้งค่าตัวแปรสถานะสตริง (String State) คอยดักจับชุดตัวเลขรหัสผ่านที่ป้อนเข้ามา 
  * รหัสผ่านที่กำหนดสิทธิ์ถูกต้องในการผ่านด่านคือ "2003" เท่านั้น
  * ระบบตรรกะต้องรองรับการป้อนข้อมูลพาสเวิร์ดขนานกัน 2 ทาง คือ การกดย้ำผ่านแผงปุ่มกดจำลอง (Keypad) บนหน้าจอ และการพิมพ์ผ่านแป้นพิมพ์คีย์บอร์ดจริงของระบบคอมพิวเตอร์ (ดักจับผ่านคำสั่ง onKeyDown)
  * เมื่อตรวจสอบพบว่ารหัสผ่านถูกกรอกเข้ามาครบ 4 หลัก จะวิ่งเข้าฟังก์ชันตรวจสอบ (Validation) ทันที หากผลลัพธ์เป็นเท็จจะสั่งเคลียร์ตัวแปรสตริงชุดนั้นทิ้งเป็นค่าว่างเปล่าทันที

2. Pastel UI & Keypad Design (การจัดวางโครงสร้างและเลเยอร์ปุ่มกด)
- เครื่องมือที่ใช้: Tailwind CSS 100%
- ตรรกะการทำงาน:
  * ออกแบบแผงปุ่มกดจำลอง (Keypad) จัดตำแหน่งโครงสร้างบล็อกสี่เหลี่ยมเรียงแถวในรูปแบบ Grid ขนาด 3x4 กึ่งกลางหน้าจอ
  * หน้าตาของตัวปุ่มกด: ใช้คลาสคุมความโค้งมนทรงกลมพาสเทล (Rounded Pastel Buttons) เลือกสลับสีแนวลูกกวาดซอฟต์ๆ มีระยะขอบ (Padding) และขนาดที่สัมพันธ์กับนิ้วมือของผู้ใช้งานระบบทัชสกรีนบนโทรศัพท์
  * ตั้งค่าระบบเลเยอร์ให้บล็อกข้อความช่องแสดงผลตัวเลข (Password Display Slot) อยู่ด้านบนสุดของแผงปุ่มกด เพื่อให้แฟนเห็นจำนวนจุดวงกลมหรือตัวเลขพาสเวิร์ดที่พิมพ์ไปแล้วชัดเจน

3. Animation & Audio Mapping (ระบบควบคุมการเคลื่อนไหวและเอฟเฟกต์เสียง)
- เครื่องมือที่ใช้: Framer Motion, ไลบรารีภายนอก (Canvas Confetti), และระบบปลดล็อกเสียง (Audio Context API)
- ตรรกะการทำงาน:
  * เอฟเฟกต์การกดปุ่มแผง Keypad: ใส่คำสั่งจำลองแรงยุบตัวยุบพองแบบนุ่มนวลชั่วขณะ (Soft Press Animation) ทุกครั้งที่มีการจิ้มหรือคลิกลงบนตัวปุ่ม
  * กรณีป้อนรหัสผ่านไม่ถูกต้อง (Failure Event): แผงปุ่มกด Keypad และช่องแสดงผลพาสเวิร์ดทั้งหมดต้องเล่นอนิเมชันสั่นสะเทือนสไลด์สลับซ้าย-ขวาอย่างรวดเร็ว (Shake Animation) เพื่อส่งสัญญาณเตือนแฟนว่ารหัสผิด ก่อนจะรีเซ็ตค่าสถานะตัวเลข
  * กรณีป้อนรหัสผ่านถูกต้อง "2003" (Success Event):
    * สั่งการรันฟังก์ชันระบบจุดพลุฉลองความสำเร็จ เรียกใช้งานชุดคำสั่งยิงเศษกระดาษพาร์ทิเคิลรูปหัวใจระเบิดฟู่ออกมาพุ่งกระจายเต็มหน้าจอ (Canvas Confetti) 
    * ตัวหน้าจอหลักจะเล่นอนิเมชันเปิดเอฟเฟกต์แสงวาบสีขาวนวลสว่างเต็มจอ (Flash Warp Effect) พร้อมกับลดค่าความโปร่งแสงค่อยๆ เฟดสลับเปลี่ยนหน้าต่างฉากหลัง ข้ามเข้าสู่หน้าจอระบบออกตั๋วเดินทางลอนดอน (Step 2) อย่างต่อเนื่องไร้รอยต่อ
    * ใช้จังหวะที่แฟนกดปุ่มตกลงสำเร็จในเฟสนี้ ทำการสั่งปลดล็อกระบบเสียงของบราวเซอร์มือถือ (Trigger Audio Context Unlock) เพื่ออนุญาตให้ระบบสามารถสตรีมเสียงเพลงพื้นหลัง (BGM) และเสียงเอฟเฟกต์เฉพาะตัวในด่านถัดๆ ไปได้โดยไม่โดนระบบความปลอดภัยของมือถือบล็อกสัญญาณเสียง
