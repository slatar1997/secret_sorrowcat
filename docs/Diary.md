### 📖 Step 4_3: Memory Diary Logic & Interaction (ตรรกะและโครงสร้างระบบสมุดภาพความทรงจำ)

1. State & Pagination Logic Flow (ระบบจัดการสถานะและตรรกะการแบ่งหน้า)
- เครื่องมือที่ใช้: React State (useState) และการจัดการข้อมูลแบบอาร์เรย์ (Array Slicing)
- ตรรกะการทำงาน:
  * ใช้ตัวแปรสถานะดักจับลำดับหน้าปัจจุบัน (เช่น currentPageIndex: number) เพื่อสั่งงานให้ตัวควบคุมสมุดรู้ว่าต้องแสดงผลหน้าใด (รองรับหน้าแรก Page 1 ไปจนถึงหน้าสุดท้าย Page 5)
  * ตรรกะการกระจายรูปภาพ (Pagination Array Mapping): นำชุดข้อมูลอาร์เรย์รูปภาพทั้งหมด 20 ใบมาผ่านฟังก์ชันตัดแบ่งข้อมูล (Slice Method) ออกเป็นชุดย่อย ชุดละ 4 รูปโดยอัตโนมัติ เพื่อนำไปกระจายตัวลงตามหน้าบล็อกตารางกระดานสมุดทั้ง 5 หน้าอย่างเป็นระบบ:
    - หน้าที่ 1 (Index 0) ➡️ แสดงไฟล์รูปภาพลำดับที่ `photo-01.png` ถึง `photo-04.png`
    - หน้าที่ 2 (Index 1) ➡️ แสดงไฟล์รูปภาพลำดับที่ `photo-05.png` ถึง `photo-08.png`
    - หน้าที่ 3 (Index 2) ➡️ แสดงไฟล์รูปภาพลำดับที่ `photo-09.png` ถึง `photo-12.png`
    - หน้าที่ 4 (Index 3) ➡️ แสดงไฟล์รูปภาพลำดับที่ `photo-13.png` ถึง `photo-16.png`
    - หน้าที่ 5 (Index 4) ➡️ แสดงไฟล์รูปภาพลำดับที่ `photo-17.png` ถึง `photo-20.png`
  * **ข้อกำหนดการแชร์สถานะข้ามด่าน (Global Asset Export):** เนื่องจากรูปถ่ายคู่ทั้ง 20 ใบนี้จะถูกนำไปใช้เล่นเอฟเฟกต์พุ่งกระจายระเบิดลอยออกมาเต็มจอพร้อมกับไอเทมอื่น ๆ ในฉากเปิดกล่องของขวัญตอนจบ (Step 4_4 Grand Finale) ตัวแปรพาธรูปภาพชุดนี้ต้องถูกจัดเก็บหรือตั้งชื่อให้เรียกใช้ต่อได้ง่ายในระดับโครงสร้างหลัก (Root Context/State)

2. Asset & UI Layout Design (การจัดวางโครงสร้างเลเยอร์สมุดภาพและโพลารอยด์)
- เครื่องมือที่ใช้: Tailwind CSS 100%
- ตรรกะการทำงาน:
  * สถาปัตยกรรมมุมมองมุมสูง (2D Top-Down Book View): จัดวาง Component สมุดบันทึกให้แบนราบไปกับพื้นหลังโต๊ะทำงาน木ในมุมมองมองตรงลงมาจากด้านบน (Top-down Perspective) ห้ามทำมุมเอียงเพื่อให้แฟนรู้สึกเหมือนก้มอ่านหนังสือจริงบนโต๊ะ รองรับการย่อขนาดแบบ Responsive ตามหน้าจอมือถืออัตโนมัติ
  * โครงสร้าง Grid หน้ากระดาษ: ภายในหน้ากระดาษแต่ละหน้าจะถูกจัดแบ่งบล็อกด้วยเลเยอร์ระบบตารางขนาด 2x2 (Grid Grid-cols-2 Gap-4) เพื่อกระจายวางรูปถ่ายสไตล์โพลารอยด์หน้าละ 4 ใบอย่างสมดุล
  * สเปกกล่องรูปโพลารอยด์ (Polaroid Component Spec): 
    - กรอบรูปใช้สีขาวครีมหนา มีเงาตกกระทบจางๆ ด้านล่าง (shadow-md) 
    - ภายในกล่องบังคับล็อกสัดส่วนรูปภาพด้วยคลาส `aspect-square` และ `object-cover` เสมอ เพื่อล็อกรูปคู่ไม่ให้เกิดอาการบีบอัด เบี้ยว ยืด หรือล้นหลุดขอบกระดาษโพลารอยด์
    - เว้นพื้นที่ขอบขาวด้านล่างของรูปถ่ายแต่ละใบไว้สำหรับแสดงกล่องข้อความบรรยายความทรงจำสั้น ๆ โดยระบุบล็อกข้อความให้ใช้ฟอนต์สายใยอบอุ่น (Font: Mali) ทั้งหมด

3. Animation & PageFlip Interaction (ระบบควบคุมการเคลื่อนไหวและการพลิกหน้า 3D)
- เครื่องมือที่ใช้: ไลบรารีคอมโพเนนต์ `react-pageflip` (หรือสถาปัตยกรรม Framer Motion คุมแกนหมุนพิกัด 3D)
- ตรรกะการทำงาน:
  * ระบบพลิกหน้ากระดาษสามมิติ (3D Page Flipping Physics): ตัวหน้ากระดาษสมุดต้องรองรับการใช้คำสั่งเลื่อนปัดนิ้วลาก (Swipe) ผ่านหน้าจอมือถือ หรือการคลิกเมาส์แล้วลากเมาส์สะบัด (Click & Drag) บนหน้าจอคอมพิวเตอร์ โดยขอบปีกกระดาษต้องเล่นเอฟเฟกต์โค้งงอบิดตามแรงดึงในมิติแกนหมุน 3D อย่างสมจริง
  * ตัวโครงสร้างกรอบสมุดโดยรวม (Bounding Box) จะต้องถูกตรึงพิกัดระนาบนิ่งสนิทอยู่ในแกนมองตรงจากด้านบน (Top-down Boundary) ห้ามหลุดโฟกัสหรือเอียงบิดตัวตามหน้ากระดาษจนเสียสมดุลสายตา
  * ระบบปุ่มกดเสริมความสะดวก (Pagination Nav Button): นอกเหนือจากการลากพลิกแผ่นกระดาษด้วยนิ้วแล้ว ให้จัดวางปุ่มกดลูกศรพาสเทลซ้าย-ขวาไว้บริเวณข้างตัวสมุด (whileTap ยุบพองตัวได้) เพื่อให้แฟนสามารถกดสั่งพลิกหน้ากระดาษไปข้างหน้า (Next) หรือย้อนกลับ (Prev) ได้อย่างอิสระ โดยตัวกระดาษจะเล่นแอนิเมชันพลิกแผ่น 3D แบบเดียวกัน
