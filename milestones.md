### Milestone 3 : Dynamic Ticket System & UI Layout Specs (ระบบตั๋วแปรผันตามชื่อและดีไซน์)

1. State & Logic Flow (ระบบจัดการสถานะและตรรกะ)
- เครื่องมือที่ใช้: React State (useState)
- ตรรกะการทำงาน:
  * ใช้ State ชื่อ passengerName สำหรับผูกกับช่องกรอกข้อมูล (Input) เพื่อรับชื่อของแฟนในรูปแบบสายอักขระ (String)
  * เมื่อแฟนกดปุ่ม "ออกตั๋ว" ระบบจะสั่งคำสั่งเปลี่ยนสถานะเพื่อส่งต่อค่า passengerName นี้เข้าไปแสดงผลบนตำแหน่งระบุชื่อของ Component ตั๋วรถไฟโดยอัตโนมัติ
  * มีการใช้ State ประเภท Boolean คอยควบคุมสถานะการพลิกการ์ดหน้า-หลัง (เช่น isFlipped) เพื่อสลับหน้าการแสดงผลเมื่อเกิดการคลิกที่ตัวการ์ดตั๋ว

2. Ticket UI Design & Layout (การจัดวางโครงสร้างและเลเยอร์หน้าจอด้วย Tailwind CSS)
- เครื่องมือที่ใช้: Tailwind CSS 100%
- ตรรกะการทำงาน:
  * กำหนดธีมสีสไตล์วินเทจพาสเทล: ตัวตั๋วใช้สีครีมกระดาษเก่า (bg-[#FDFBF7]) และเส้นขอบใช้สีส้มชานมพาสเทลอบอุ่น (border-[#FFA07A])
  * โครงสร้างหน้าตั๋ว: กำหนดรูปทรงสี่เหลี่ยมผืนผ้าแนวนอน ใช้เส้นประคั่นสีน้ำตาลแบ่งสัดส่วนพื้นที่การ์ดออกเป็น 2 ฝั่ง คือ ฝั่งเนื้อหาหลัก (Main Area) ด้านซ้าย 70% และฝั่งตอนฉีกตั๋ว (Stub Area) ด้านขวา 30%
  * การทำรอยเว้าตั๋วรถไฟ: ใช้กล่องมาร์กทรงกลมเปิดคลาส rounded-full จัดพิกัดแบบ absolute ขยับดันหลบเข้าขอบจอทั้งฝั่งซ้ายและขวา เพื่อเจาะรูเว้าครึ่งวงกลมให้เหมือนตั๋วรถไฟจริง
  * ระบบจัดระเบียบข้อมูลบนตั๋ว (Ticket Content Mapping):
    * ฝั่งตั๋วหลัก (ซ้าย 70%): โลโก้วงกลมวาดด้วย CSS พร้อมตัวอักษร "LONDON TRANSIT" (Font: Itim) จัดข้อมูลเป็นโครงข่าย 2 คอลัมน์สวยงาม ตรงช่อง Name แสดงค่าแปรผันจากตัวแปร [passengerName] ด้วยตัวหนาสีน้ำตาลเข้ม (Font: Mali) ส่วนช่องอื่นล็อกค่าตายตัว (Destination: Together / Start Date: 19 June 2026 / Valid Until: No expired) และมีตัวอักษรจางๆ คำว่า "Love Express" ทำหน้าที่เป็นลายน้ำด้านหลัง
    * ฝั่งตอนฉีกตั๋ว (ขวา 30%): แสดงข้อความ Class: 1ST (ใส่กรอบสี่เหลี่ยมมนสีส้ม ตัวอักษรขาว) / Seat: Heart / Time: 20.26 / Number: 190626 และด้านล่างสุดสร้างเส้นบาร์โค้ดจำลองจากการจัดเรียงกล่อง Div สีดำหนา-บางสลับความกว้างขัดกันถี่ๆ เพื่อสร้างความสมจริงโดยไม่ต้องโหลดรูปภาพ
    * ฝั่งด้านหลังการ์ด: ถมสีกระดาษแบบเดียวกัน พร้อมระบุข้อความกึ่งกลางระบุว่า "ticket for love"

3. Animation & Interaction Mapping (ระบบควบคุมการเคลื่อนไหวและเอฟเฟกต์)
- เครื่องมือที่ใช้: Framer Motion และระบบจัดการเสียง (Audio JavaScript API)
- ตรรกะการทำงาน:
  * แอนิเมชันตั๋วเลื่อน: เมื่อกดปุ่มออกตั๋ว ให้ตัวการ์อร์ตั๋วเล่นอนิเมชันเลื่อนสไลด์ขึ้นมาจากขอบจอด้านล่าง (ขยับพิกัดแกน Y จากล่างขึ้นบน) ค่อยๆ โผล่ขึ้นมาจากตู้ตั๋วจำลอง ควบคู่กับการสั่งรันไฟล์เสียงเครื่องพิมพ์ตั๋ว (ticket-print.mp3) ทันที
  * แอนิเมชันพลิกการ์ด 3D (3D Card Flip): ครอบตัวตั๋วด้วยกล่องระบุค่าทัศนียภาพความลึก (Perspective) และเปิดคำสั่งซ่อนพื้นผิวด้านหลัง (backface-hidden) เมื่อแฟนคลิกที่ตั๋ว ให้สั่งคำสั่งเคลื่อนไหวหมุนแกนแนวนอน (rotateY: 180) เพื่อพลิกสลับหน้าตั๋วไปแสดงข้อความด้านหลังแบบสมจริง โดยโครงสร้างไม่บิดเบี้ยว

### Milestone 4 : Cinematic Sliding View (ระบบอนิเมชันรถไฟวิ่งผ่านหน้าต่าง)

1. State & Logic Flow (ระบบจัดการสถานะและตรรกะ)
- เครื่องมือที่ใช้: React State และ Callback ของระบบอนิเมชัน
- ตรรกะการทำงาน:
  * ควบคุมการสลับเข้าฉากนี้ด้วยสถานะสเต็ปหลัก (เช่น currentStep === 'cinematic') หลังจากตัวตั๋วใน Step 2 เล่นกระบวนการเสร็จสิ้น
  * ไม่ใช้ระบบตั้งเวลาแบบวินาทีตายตัว (Timer) เพื่อสั่งเปลี่ยนหน้าจอ แต่จะใช้ฟังก์ชันดักจับแอนิเมชันเคลื่อนภาพเสร็จสมบูรณ์ (onAnimationComplete) ของไลบรารีแอนิเมชันเป็นตัวส่งสัญญาณ (Trigger) ข้ามไปด่านถัดไป (Desk Scene / โต๊ะทำงานหลัก) โดยอัตโนมัติ เพื่อป้องกันปัญหาภาพสไลด์ไม่จบแล้วฉากตัดก่อน
  * ระบบ Transition ข้ามฉากปลายทาง: เมื่อภาพเลื่อนสิ้นสุดลง ให้สั่งเปิดสวิตช์ Effect ผสมผสานระหว่างการเลือนหาย (Fade out) + การเบลอภาพ (Blur) + แสงวาบสีขาว (White Flash) ผ่านการเปลี่ยนค่า Opacity และ Filter เพื่อตัดเข้าด่านถัดไปแบบนุ่มนวลที่สุด

2. Asset & UI Layout Design (การจัดวางโครงสร้างและเลเยอร์หน้าจอ)
- เครื่องมือที่ใช้: Tailwind CSS 100%
- ตรรกะการทำงาน:
  * แยกโครงสร้าง Layer หน้าจอออกเป็นสองชั้นเด็ดขาดด้วยระบบ Z-Index:
    * เลเยอร์คงที่ด้านบน (Static Mask Layer): วาดภาพพื้นหลังห้องโบกี้รถไฟ (Train Cabin) และจัดวางองค์ประกอบกรอบหน้าต่างรถไฟไว้กึ่งกลางจออย่างนิ่งสนิท 100% ห้ามสั่งเคลื่อนไหวตัวหน้าต่างนี้เด็ดขาด โดยเปิดคลาสบังคับซ่อนขอบวัตถุล้น (overflow-hidden) บนตัวกรอบหน้าต่างเพื่อทำหน้าที่เป็นหน้ากาก (Mask) บังภาพวิวส่วนเกิน
    * เลเยอร์เคลื่อนไหวคู่ขนานด้านล่าง (Moving Background Layer): สร้างกล่อง Flexbox แนวนอนแนวยาว (flex-row) ซ่อนไว้ด้านหลังกรอบหน้าต่างรถไฟ นำไฟล์ภาพวิวสีน้ำลอนดอน 3 ภาพมาจัดวางต่อแถวกันตามลำดับ คือ london-1.jpg (Big Ben) -> london-2.jpg (แม่น้ำเทมส์) -> london-3.png (Tower Bridge) ขยายขนาดความกว้างรวมให้ล้นออกไปนอกเขตกรอบหน้าต่างเตรียมพร้อมรอการเคลื่อนที่

3. Animation & Interaction Mapping (ระบบควบคุมการเคลื่อนไหวและเอฟเฟกต์)
- เครื่องมือที่ใช้: Framer Motion
- ตรรกะการทำงาน:
  * การเคลื่อนที่ของภาพวิว: สั่งอนิเมชันไปที่กล่อง Container แถบภาพวิวด้านหลังเท่านั้น โดยปรับค่าระยะเยื้องพิกัดแกนนอนจากพิกัดเริ่มต้นศูนย์ ไปสู่ค่าติดลบทางซ้าย (transform: translateX จาก 0 ไปค่าติดลบ) เพื่อดันแถวภาพให้ไหลผ่านช่องหน้าต่างจากขวาไปซ้าย
  * ห้ามใช้คำสั่งขยับ Layout จำพวกการปรับค่า Margin หรือตำแหน่ง Position Absolute พิกัด Left/Right เด็ดขาด เพื่อป้องกันไม่ให้บราวเซอร์มือถือเกิดอาการกระตุกหรือเฟรมเรตตก
  * การควบคุมความเร็ว: กำหนดรูปแบบการผ่อนน้ำหนักแอนิเมชันเป็นแบบความเร็วสม่ำเสมอเท่ากันตั้งแต่ต้นจนจบ (ease: "linear") เพื่อจำลองความเร็วของรถไฟที่กำลังแล่นด้วยความเร็วคงที่ โดยตั้งค่าระยะเวลาลากยาว (Duration) ไว้ที่ประมาณ 10–20 วินาที เพื่อสร้างมู้ด Cinematic
  * ตรรกะจังหวะการหยุด: เมื่อแถบภาพเลื่อนมาจนสุดระยะพิกัดของภาพใบสุดท้าย (london-3.png สะพาน Tower Bridge) ระบบอนิเมชันจะสั่งให้ตัวภาพหยุดนิ่งตรึงอยู่กึ่งกลางกรอบหน้าต่าง และสั่งสั่งการดีเลย์ค้างหน้าจอเอาไว้เป็นเวลา 0.5–1 วินาที เพื่อให้แฟนได้อ่านและมองภาพสถานีปลายทางชัดเจน ก่อนที่จะส่งสัญญาณไปรันฟังก์ชันเปลี่ยนฉากในเงื่อนไข onAnimationComplete ตามตรรกะฝั่งข้อที่ 1

## Milestone 5: Step 1A - Central Table & Vinyl Station (เฟสโต๊ะหลักและเครื่องเล่นแผ่นเสียง)
เป้าหมาย: สร้างหน้าเลย์เอาต์โต๊ะทำงานมุมมอง Top-down รองรับมือถือ และวางระบบเครื่องเล่นเสียง 12 เพลง

งานที่ต้องทำ:

สร้างคอมโพเนนต์ MainTable.tsx จัดวางสิ่งของแบบ Absolute Positioning คุมสเกลแบบ Responsive

พัฒนาคอมโพเนนต์ VinylStation.tsx วาดเครื่องเล่นแผ่นเสียงสีน้ำตาลและแผ่นไวนิลสีดำด้วย CSS 100%

ใช้คลาส object-cover และ rounded-full จัดกึ่งกลางภาพหน้าปกคู่ (cover-01 ถึง cover-12) ด้วยวิธี Absolute Centering

เขียนตรรกะลากวาง (Drag & Drop) และคลิกเพื่อสลับแผ่นเสียง โดยแผ่นเก่าต้องวิ่งกลับกล่องอัตโนมัติ

วางระบบควบคุมเสียงไว้ที่ระดับบนสุดของแอป (Global Audio) พร้อมผูกตรรกะปุ่ม Play / Pause / Stop และแอนิเมชันก้านหัวเข็ม

Completion Criteria (เกณฑ์การผ่าน):

[ ] รูปคู่ดุมกลางแผ่นเสียงต้องกลมดิ๊ก 100% และคงสัดส่วนเดิมเสมอ ไม่ยืด ไม่บี้แบน ไม่ว่าไฟล์ภาพดิบจะเป็นแนวตั้งหรือแนวนอน

[ ] เมื่อเปลี่ยนแผ่นเสียง เพลงเก่าต้องหยุดทันที ก้านหัวเข็มหมุนมาแปะขอบแผ่น ดีเลย์เล็กน้อย แล้วแผ่นเสียงต้องหมุนพร้อมเพลงใหม่เล่น

[ ] เพลงต้องเล่นคลอเป็นพื้นหลังอย่างต่อเนื่อง แม้แฟนจะกดปิดหน้าต่างเครื่องเล่นแผ่นเสียงเพื่อไปทำกิจกรรมอื่นบนโต๊ะ

[ ] ปุ่มควบคุมทำงานถูกต้อง: Pause = หยุดเพลง+แผ่นหยุดหมุน+เข็มค้างไว้, Stop = หยุดเพลง+รีเซ็ตเวลา 0:00+แผ่นหยุดหมุน+เข็มดีดกลับแท่น
